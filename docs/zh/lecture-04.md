# 04 — 一起精读一篇 ModelScope 模型卡

> 本课程第四讲视频脚本。约 11 分钟。
> 建议在看过第二、三讲并大致掌握 220 个词、15 个句型之后再看。

---

## 课程脚本

大家好,这是第四讲。看完前三讲的海外学员,这时候已经在背 220 个领域词,也认识了 15 个语法句型。但学到这一步并不算完事。真正的工作从这一讲开始。所以这一讲,我们就带海外学员真的去 ModelScope 上,挑一篇模型卡,一段一段地读。真正的实战。

我们设个场景。海外学员听说 Qwen3 出新版本了,打开 ModelScope。URL 大概是 `modelscope.cn/models/Qwen/Qwen3-235B-A22B`。页面打开,第一段大致是这样:

> Qwen3-235B-A22B 是基于 Transformer 架构的大语言模型,支持文本生成、对话、推理等任务。该模型采用 MoE 设计,总参数量为 2350 亿,激活参数为 220 亿,可用于多种自然语言处理场景。

对中文母语朋友来说,这一段五秒就读完了。但对一个学了三个月中文的海外学员来说,这一段的处理过程大概是这样的 —— 我带你看一遍。

**第一句前半:** `Qwen3-235B-A22B 是基于 Transformer 架构的大语言模型`。

Qwen3-235B-A22B 是模型名,跳过。下一个字是 「是」 —— 海外学员第一周就背会了。然后 「基于」 —— 这是 15 句型里他刚学的一个,等于英文的 "based on"。然后 「Transformer 架构」 —— 「架构」 这个词,韩国学员用韩语汉字音读直接是 "가구",但意思他在领域词汇里背过,等于英文 "architecture"。英文学员则是把 「架」(frame)和 「构」(structure)两个字拆开,合起来识别成 "architecture"。然后 「的」 —— 助词。然后 「大语言模型」—— 海外学员的脑内处理是:大 = big,语言 = language,模型 = model,合起来就是 LLM。

合起来:**"Qwen3-235B-A22B 是基于 Transformer 架构的大语言模型。"** 这一小段他大概花了一两分钟。

**第一句后半:** `支持文本生成、对话、推理等任务`。

「支持」 —— 韩国学员认识为 "지지",但课程里他已经看到这个词在中文 IT 上下文是 "支持/支援"(英文 support)。「文本生成」—— 文本是 text,生成是 generate。「对话」—— 对话,这个直接亮。「推理」—— 推理 = inference,这个领域词他第一周就背会了。「等任务」—— 等是 "etc.",任务是 "task"。

**第二句:** `该模型采用 MoE 设计,总参数量为 2350 亿,激活参数为 220 亿,可用于多种自然语言处理场景`。

「该模型」—— 「该」 是书面语的 "this/that"。「采用」—— 海外学员可能没见过这个词,他查一下,记下 "adopt"。「设计」—— design,直接亮。所以前半句:"This model adopts the MoE design."

「总参数量为 2350 亿」—— 「总」 = total,「参数量」 = parameter count,「为」 = is,「亿」 = 100 million。结果:"Total parameter count is 235 billion." 「激活参数为 220 亿」—— activated parameters = 22 billion。海外学员看到这一组数字会反应过来 "啊这是 MoE,稀疏激活"。

「可用于多种自然语言处理场景」—— 「可用于」是 15 句型之一,「多种」 = various,「自然语言处理」 = NLP,「场景」海外学员可能查一下,记下 "scenario / use case"。

整段合起来:

> Qwen3-235B-A22B is a large language model based on Transformer architecture, supporting text generation, dialogue, inference, and other tasks. The model adopts MoE design, with a total parameter count of 235 billion and 22 billion activated parameters, and can be used for various NLP scenarios.

这一段,中文母语者五秒读完。海外学员花了大概 20 到 30 分钟。但这 20 到 30 分钟里发生的事情,是一种从被动消费翻译稿件,到主动从一手资料里直接取信息的能力转变。这件事一旦发生,他下一篇 model card 大概只要 10 分钟,再下一篇可能只要 5 分钟,一个月之后他读你写的中文段落速度,会接近他读英文 README 的速度。

那这段录像我做给中文母语朋友的意义是什么?三件事。

**第一件事**,你写 model card、写知乎答案、写 CSDN 博客的时候,请尽量用清晰、简洁、固定结构的句型。「该模型采用」、「基于...架构」、「用于...场景」 —— 这些固定结构对海外学员是友好的,他们能直接套上他们学过的 15 个句型读懂。如果你为了文笔好看而写得太花哨,海外学员会卡在那里。这不是要你委屈自己,是请你知道:**清晰简洁的中文,正在成为一种全球可读的中文**。

**第二件事**,你的术语用词请尽量稳定。如果你这一段写 「推理」,下一段写 「推断」,再下一段写 「inference」,海外学员的词汇匹配会断三次。中文母语者读起来差别不大,但对查字典的海外学员每一次切换都是成本。

**第三件事**,如果你看到一个海外开发者在评论区或者 GitHub Issue 里把你的术语用得有点僵硬 —— 比如他用 「采用」 用得过度,或者每句话都从 「该模型」 开始 —— 不要笑。那是他严格按照学过的 15 句型写出来的结果,就像我们用英文写邮件时也会显得过于规整。请理解,他在用一种不擅长的语言对你写的代码表达兴趣。

最后,如果你自己反过来想读懂英文一手资料,这一讲整个分析流程也成立 —— 找一篇 PyTorch 的 README,挑前一段,逐句拆,把不会的查字典并归类(术语 vs 通用词 vs 句型),然后跟 DeepL 比对。一个月后你读 PyTorch README 就跟读中文 README 没差。

四讲到这里完整结束了。看到这里的海外学员,你已经具备了开始的全部条件。看到这里的中文母语朋友,谢谢你愿意陪一段不是写给你的课程从头听到尾 —— 你的语言、你的写作、你的开源项目,正是我们这门课最重要的教材。

谢谢大家。
