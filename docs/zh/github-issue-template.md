# GitHub Issue 中文模板集

> 适用于向 DeepSeek、Qwen、GLM 等中文开源项目提交 Issue / Discussion / PR 的可直接复用模板。
> 仅需将 `[]` 标记的内容替换为自己的实际情况。

---

> **致中文母语贡献者**：本文档面向非中文母语的外国开发者，旨在帮助他们以符合中文社区习惯的方式提交 Issue。如果您看到不自然的措辞，请提交 PR。

---

## 1. Bug 报告 (Bug Report)

```markdown
## 问题描述

[一句话概括 bug]

## 环境信息

- 操作系统：[例：Ubuntu 22.04 / macOS 14.5 / Windows 11]
- Python 版本：[例：3.11.7]
- 框架版本：[例：torch 2.3.0, transformers 4.42.0]
- GPU：[例：NVIDIA A100 40GB / RTX 4090 / 无 GPU]
- CUDA 版本：[例：12.1]
- 模型版本：[例：DeepSeek-V3-Base]

## 复现步骤

1. [第一步]
2. [第二步]
3. [第三步]

## 预期行为

[期望发生的行为]

## 实际行为

[实际发生的情况]

## 报错信息

\```
[完整的堆栈跟踪 / 错误日志]
\```

## 已尝试的解决方案

- [尝试 1]
- [尝试 2]
- 已查阅相关 issue：#[相关 issue 编号]，但未找到解决方案。

## 补充信息

[截图、额外上下文、可疑环节等]
```

## 4. 性能 / OOM 问题

```markdown
## 问题描述

在运行 [模型名] 的推理时遇到显存不足（OOM）问题。

## 环境信息

- GPU：[GPU 型号 + 显存]
- CUDA 版本：
- 框架版本：
- 模型：[模型名 + 参数量]
- 输入长度：[输入 token 长度]
- 批大小：[batch size]

## 报错信息

\```
torch.cuda.OutOfMemoryError: CUDA out of memory. Tried to allocate ...
\```

## 已尝试的优化

- [ ] 减小 batch size 至 [数值]
- [ ] 启用梯度检查点（gradient checkpointing）
- [ ] 使用混合精度（fp16 / bf16）
- [ ] 启用模型量化（[INT8 / INT4]）
- [ ] 减小最大序列长度
- [ ] 使用 CPU offload

## 期望

请问是否有推荐的配置可以在 [显存大小]GB 显存上运行该模型？
```

## 8. 致谢 + 追问

```markdown
感谢您的回复！

按照您的建议，我 [尝试的内容]，结果 [结果].

不过还有一个问题：[追加的问题].

请问 [具体问题]？
```

## 常用单句

| 中文 | 用途 |
|------|------|
| 您好，我想反馈一个问题 | 问候 + 报告开头 |
| 我遇到了 ... | 报告问题 |
| 可以稳定复现 | 说明可复现 |
| 无法稳定复现 | 说明不可复现 |
| 我尝试了 ... 但没有效果 | 报告失败的尝试 |
| 请帮忙看看 | 请求查看 |
| 感谢您抽时间查看 | 致谢 |
| 如需更多信息请告知 | 补充说明 |
| 相关代码如下 | 引出代码 |
| 我也遇到了同样的问题 | 表明相同情况 |
| 同+1 | 同意 |
| 这个问题还没解决 | 状态确认 |
| 已提交 PR：#[编号] | 关联 PR |
| 将在下一版本中修复 | 维护者回复 |

## 风格指南

- 中文开源社区惯用敬语 (`您`)，比 `你` 更稳妥。
- 比起直白命令，多用 `请` `麻烦` `感谢` 等委婉用语。
- 英文技术词汇可以保留英文 (PyTorch、transformer、batch size 等)。
- 代码块使用 ` ``` ` (与英文社区相同)。
- 慎用表情符号。中文开源 issue tracker 整体语气比英文圈更克制。
