## 课程脚本

大家好,这是第三讲。在第二讲里我说,海外学员要在一个月内背完 220 个领域词。这一讲,我想说的是 —— 词学完了,接下来怎么读句子?也就是,为什么 15 个语法句型就够用了。

先问一个问题。屏幕前的中文母语朋友,你的英语口语怎么样?诚实回答。如果你说不太行,那下一个问题 —— 你能读懂英文 README 吗?大部分中国开发者会在第一个问题上说 "不太行",在第二个问题上说 "能读"。同一个人。

这件事如果细想其实挺奇怪的。一个不能跟人用英语对话的人,为什么能读懂英文技术文档?答案出乎意料地简单 —— **口语语法和阅读语法实际上是两个不太重叠的领域**。口语里频繁出现的句型在那里,阅读里频繁出现的句型在另一个地方,两者的交集比一般人以为的要小得多。

那把这件事换到中文上。中文口语里最重要的语法之一,是语气助词 —— 吧、啊、呢、嘛 之类的。对外国学员来说,这是个特别陌生的概念,他们的母语里没有完全对应的东西。这些字挂在句末,微调情绪。"你来吧" 是 "来一下嘛","你来啊" 是 "你倒是来啊","你来呢" 是 "那你呢、你怎么来"。看起来差不多,语气差很多。掌握这些是口语语法的核心。

那这些语气助词会出现在 ModelScope 的 model card 里吗?不会。不会有 「吧」,不会有 「啊」。原因很简单 —— 技术文档不是在传递说话人的情绪,是在传递事实。语气助词在技术文档里几乎不出现。

那么模型卡里真正高频的语法是什么?给海外学员先看五个。

第一,**把字句**。把 X 处理成 Y。`把模型转换为 ONNX 格式` —— 把模型转换成 ONNX 格式。代码说明里出现的频率非常高。

第二,**将** —— 把字句的书面语版本。在官方文档、论文里,「将」 是标准,而不是 「把」。`将参数初始化为 0` —— 将参数初始化为 0。

第三,**用于** —— 用于做某事。`该模型用于文本生成` —— 几乎是每一篇 README 第一句话的标配。

第四,**基于** —— 基于某物。`基于 Transformer 架构` —— 模型介绍里基本上不可能不出现。

第五,**对……进行** —— 对 X 进行 Y。`对模型进行量化` —— 这是真正的高频结构,可以当成技术语境的固定表达。

光这五个,model card 大概就读懂一半了。再加上是……的(强调)、不/没(否定)、可以/能(能力)、如果……就(条件)、通过(借助)、需要(需要)、支持(支持)、被(被动)、连动句(两个动词连用)、得(程度补语)。这十个加起来就是十五个。这十五个把模型卡和 README 的覆盖率推到 80% 以上。

那这十五个能用来口语吗?不太行。如果一个海外学员只用这十五个句型说话,听起来会特别像中文学习机器人。但那不是目标。海外学员的目标不是跟你聊天,是读你写的 model card。所以在口语里偏低效的结构,反而成了精确的阅读工具。

还有个好消息要给海外学员 —— 中文的语序是 SVO,跟英文一样。我吃饭 = I eat rice。日韩语序是 SOV(我饭吃),所以日韩学员一开始会觉得别扭,但只要平时看英文,他直接套英文的语序就够用。一个月内每天读一段 model card,语序就自然内化了。语序不靠背,靠暴露。

那我现在反过来,把这一段讲给屏幕前的中文母语朋友。

下次你看到一个海外开发者在 GitHub Issue 里写中文,句子比较直,没有 「吧」 没有 「啊」,听起来有点干,你可能会觉得 "这中文好硬"。但请理解 —— 他是用 15 个核心句型读懂你写的代码、写出他能用的中文。他不是要表达情感,他是在写技术报告。请别把他这种语气误认为冷淡或者不礼貌。这就是阅读优先策略训练出来的语言形态。

第二件事,如果你自己反过来要快速读英文一手资料,这套策略也成立 —— 不要去背英语口语高频句,直接攻技术写作里的高频结构。被动语态、it is + adj. + that、given that、in order to、provided that 这些固定结构,是英文 README 和论文的语法骨架。换种说法,我们让海外学员先攻 「把/将/用于/基于/对...进行」 五个 —— 这件事在英文里是对应的。

下一讲,我们带海外学员用这 220 个词加 15 个句型,精读一篇 ModelScope 上的模型卡。真正的实战开始。

谢谢大家。
