# 学习指南 03 — 海外学员的 15 句型精通(从入门到熟练)

> 第三讲(视频)的延伸阅读。给中文母语者看的版本。
> 假设你已读过 [guide-00 零基础起点](guide-00-foundations.md) 和 [guide-02 词汇精通](guide-02-vocabulary.md)。
> 这一份告诉你 「海外学员到底是把哪 15 个句型练到自动化的」,以及为什么这 15 个能覆盖 80% 的模型卡。
> 通读约 30 分钟。

## 一、海外学员要解决的核心难题:口语语法 vs 阅读语法

中文母语者从小学到大,语法是一体化习得的 —— 你不会把 「口语语法」 和 「阅读语法」 分开。但海外学员目标只是阅读,这一前提让他们必须做一个看似奇怪的取舍:**忽略口语语法,只攻阅读语法**。

举个例子。中文口语里最重要的一组结构是**语气助词** —— 吧、啊、呢、嘛 这四个字。

```
你来吧 —— 软劝
你来啊 —— 强调
你来呢 —— 反问
你来嘛 —— 撒娇式坚持
```

中文母语者识别这四个字的微妙差别完全没难度,但对海外学员是噩梦。HSK 1-2 级就开始教,要花几个月才能熟练。

**而它们在 ModelScope 模型卡里的出现率几乎是 0%。**

为什么?因为技术文档不是表达说话人的情绪,是传递客观事实。语气助词的本质是承载情绪,所以它们在 model card、论文、README、GitHub Issue、知乎答案里几乎不出现。

这意味着 —— 中文口语最核心的一块,在我们的课程里优先级是 0。海外学员可以完全跳过这一块,直接攻 「真正会出现在他们想读的资料里」 的句型。

## 二、海外学员真正攻打的 15 个核心句型

我做了一个统计:把 5 篇真实的中文 MLOps 模型卡的所有句子做语法标注,**15 个句型覆盖了 80% 以上**。

### 核心 5 个(覆盖 50%)

| 句型 | 中文例子 | 英文对应 |
|------|---------|---------|
| 把 / 将 | 将模型量化为 INT4 | quantize the model to INT4 |
| 用于 | 用于文本生成 | used for text generation |
| 基于 | 基于 Transformer 架构 | based on Transformer |
| 对...进行 | 对模型进行量化 | perform quantization on the model |

中文母语者读这些只觉得 「正常的技术写作」,但对海外学员这是要刻意分解的句型骨架。

### HSK 体系把这些排到很靠后

值得一提的是,HSK 课程把 「把 / 将 / 基于 / 对...进行 / 用于」 大多排在 4-5 级以后。也就是说,通过 HSK 路径要 2-3 年才能学到我们让海外学员第一周就开始的内容。这就是 「按目标重新排序」 带来的时间压缩。

### 再加 10 个(覆盖到 80%)

是...的 / 不-没区分 / 可以-能区分 / 如果...就 / 通过 / 需要 / 支持 / 被 / 连动句 / 得字程度补语。

这十个对中文母语者也都是 「日常技术写作的常见结构」,但每一个对海外学员都要单独学。

## 三、海外学员怎么学这些句型(详细工作流)

### 第一步:认识模板,不背死规则

学员看到 「把模型转换为 ONNX 格式」 时,我们给他的拆解是:

```
[主语]   [把]    [对象]    [动词]    [补语]
[省略]   [把]    [模型]    [转换]    [为 ONNX 格式]
```

学员不背 「把字句的语法规则」,而是背这个模板。然后他把同一模板套到 6-7 个例句上,例如:

- 把数据加载到内存
- 把权重保存到 checkpoint
- 把模型部署到 GPU
- 把输入序列截断为 512

到第 5 个例句时,模板已经牢固。模型卡里再遇到把字句,他就是在做模板匹配,不是在解析语法。

### 第二步:每天一段真实材料练习识别

学员每天读一段真实模型卡(50-100 字),手动标注每句话用了 15 句型里的哪几个。一段大致会用到 5-7 个,比如:

```
该模型基于 Transformer 架构 ………… 基于
采用 MoE 设计 ………………………………… 普通主谓宾
可用于多种推理场景 ……………………… 用于(可+用于变形)
通过对模型进行 INT4 量化 …………… 通过 + 对...进行
显存占用减少 75% …………………………… 普通句
支持长上下文 ………………………………… 支持
需要至少 80GB 显存 ……………………… 需要
```

一段标 7 个,做 30 段,15 句型基本就肌肉记忆了。

### 第三步:遇到变形不慌

学员遇到 「适用于」「可用于」「专用于」「主要用于」 这种 「用于」 的变形时,我们告诉他:**核心句型 + 修饰**。

适用于 = 适合 + 用于
可用于 = 可以 + 用于
专用于 = 专门 + 用于
主要用于 = 主要 + 用于

15 句型 + 几个修饰词 ≈ 50 个高频复合结构。

## 四、给中文母语朋友的三件事

### 第一件:你写技术文时的句型选择直接影响海外读者

如果你写中文 model card,**有意识地用 「基于」「用于」「对...进行」 这种结构**(其实你也本来就会用),海外学员能秒解析。

但如果你为了避免重复,在同一篇里轮流用 「该模型基于 Transformer」「这个模型采用 Transformer 架构」「我们用 Transformer 搞了个模型」 —— 中文读者觉得文笔活泼,但海外学员的句型识别会断三次,每一次都要重新解析。**风格统一是阅读体验的隐性成本**。

具体建议:
- 介绍段:固定用 「是基于...的...,支持...,用于...」 模板
- 用法段:固定用 「首先 / 然后 / 最后」 顺序词
- 性能段:固定用 「对...进行...,...提升 / 减少 N%」 模板
- 协议段:固定用 「本模型采用 ... 协议发布」

### 第二件:海外学员的硬中文是 15 句型的精确堆叠

如果你看到一个海外开发者写中文写得 「太规整」「过于书面」 —— 比如他每句话都是 「该模型...」「本工具...」「对 X 进行 Y」 —— 不要笑。那是他刚学完 15 句型,把它们全部用上的样子。这就跟我们刚学英文时只会用一两个固定句型,过几个月之后才松弛下来一样。

### 第三件:中文母语者反向用 15 句型思路读英文

英文一手资料(arXiv / PyTorch README / Hacker News)的语法骨架其实也是 15 个左右的高频结构:

| 中文 | 对应英文核心句型 |
|------|----------------|
| 把 / 将 | convert X to Y / transform X into Y |
| 用于 | be used for / be designed for |
| 基于 | based on / built on top of |
| 对...进行 | perform Y on X / apply Y to X |
| 通过 | through / via / by means of |
| 需要 | requires / needs |
| 支持 | supports |
| 被 | be V-ed (passive voice) |
| 如果...就 | if X / given X / provided that |
| 是...的 | it is Y that... / X is the Y |
| 连动句 | use Python to train / download and deploy |
| 得 | -ly adverb / so V that |

掌握这 12-15 个英文句型 + 200 个英文领域词根 = 4 个月内你能读 PyTorch / vLLM / Triton 的 README。这正是我们让海外学员对着中文做的事情,反过来。

具体英文核心 5 句型的例子:

```
英文 1: it is X that ...        (对应中文 「是...的」)
英文 2: by V-ing                 (对应中文 「通过」)
英文 3: in order to V            (对应中文 「为了」)
英文 4: provided that            (对应中文 「如果」)
英文 5: with respect to X        (对应中文 「对...进行」 部分语义)
```

这五个句型在 PyTorch 的 README、Hacker News 评论、ICML 论文里反复出现,跟我们让海外学员攻的中文 5 句型在覆盖率上是同一个量级。
