# 开场课 — 为什么会有外国开发者认真学中文

> 本课程的开场视频脚本。约 12 分钟。
> 与幻灯片同步阅读。

---

## 课程脚本

大家好,这里是 MLOps Language Curriculum,一个开源的语言学习课程。

跟一般的中文学习课程不一样,这个课程不是教你说中文,也不是教中文母语者学外语 —— 它是为**外国 MLOps 工程师**写的,目标是让他们在四个月内能够直接读懂中文的 MLOps 一手资料。

所以如果你是中文母语者,正在看这个视频,我得先说一句:严格来说,你不是这个课程的目标用户。但如果你愿意花十分钟听完,你会知道为什么会有韩国和欧美的开发者在认真地学中文 —— 不是为了说,而是为了读。这件事本身,我觉得挺值得讲一下。

你知道这是什么吗?这是 Hugging Face 的首页。我刚开始接触 LLM 的时候,模型分享的入口基本上就是 Hugging Face 这一个地方。现在它当然还是核心平台,但氛围已经不太一样了。我后来想了想,为什么当时模型分享会那么集中在一个地方?大概是因为发布模型的人想把模型放到英文用户最多的地方,而用户也只需要看一个地方就够了 —— 这两边的需求刚好在英文这一侧重合,所以才有了 Hugging Face 这样的中心。

正因如此,我刚做 LLM 那会儿,英文几乎就是 ML 资料的通用语。arXiv 是英文,model card 是英文,GitHub README 是英文。会读英文,就等于能拿到全世界几乎所有的技术信息。当时这话是没毛病的。新论文用英文出,新框架用英文写文档,翻译成中文或韩文要等好一阵子。所以读得快英文,本身就是一种信息优势。

所以今天我们熟知的那些 AI 公司 —— OpenAI、Google DeepMind、Anthropic、Meta AI —— 从某种意义上,都是 "英文事实上等于全球标准" 那个时代的产物。

然后发生了一件大事。中国的开源 LLM 爆发了。2024 年底 DeepSeek-V3 发布,2025 年初 DeepSeek-R1 又跟上,过去一直被英文圈大厂温和带着走的 LLM 生态,从中国模型登场的那一刻起,平衡开始倾斜。

在这个过程里,行业的每一个角落都发生了很多细小的变化。但我最关注的一个变化是:**资料的重心开始动摇了**。换句话说,一手信息不再只从英文里出来。

所以你现在看会发现,事情挺有意思的。DeepSeek 团队的中文 model card 写得比英文版还详细。Qwen 团队在 ModelScope 上发的中文文档,比英文 README 信息量更大。知乎上,一些大模型公司的工程师亲自下场跑 troubleshooting 的回答。CSDN 和掘金,每天都在出英文圈根本看不到的实现踩坑笔记。这些内容要被整理成英文,常常得等好几天,有时候要几个星期。

这种情形,在 OpenAI、DeepMind 那个时代刚起步的时候,是想都想不到的。如果当年真有人能从非英文渠道挖出一手信息,那时候我们叫他 "信息天才"。今天能做到这件事的人,我们就叫他 "工程师"。

称呼怎么变化,本身就在暗示时代怎么变化。那么这个变化是怎么发生的?难道一两年时间里,中国的研究者突然就变聪明了?当然不是。一个领域的研究水平,是不可能在一代人之内突然翻几倍的。那到底是什么造成了这个变化?

我觉得是这一点 —— **一手资料生产的语言比例,发生了戏剧性的变化**。没有任何一个领域,比 LLM,尤其是开源 LLM,经历过更剧烈的一手资料语言多元化。这大概是因为这个领域的论文、模型、代码、社区帖子几乎是同时产出的,信息周期非常短。

代表这种戏剧性变化的几个画面,我准备了几个,接下来一起看。

首先是 **ModelScope**。这个不用我多介绍,在座的中国朋友应该都熟,阿里巴巴做的中文模型托管平台。但你可能没意识到,对很多外国工程师来说,ModelScope 是一扇至今还没打开过的门。一方面是没听说过,另一方面是听说过也不敢点进去 —— 因为页面是中文的。能直接读懂这些中文 model card,意味着外国工程师能在几分钟内拿到那些没被翻成英文的 benchmark 和量化选项。这件事的钥匙,就是中文一手阅读能力。

然后是 **知乎**。对中国开发者来说,这是日常工具,几乎不用解释。但对外国工程师来说,知乎是一个传说中的宝藏 —— 而因为只有中文,基本进不去。设想一下,有人在用 vLLM 跑某个模型遇到 OOM,英文 GitHub Issue 里没找到解。从他的角度看,这就是卡死了。但换个角度看,他不是卡死,他只是用英文搜了一遍。答案早就在某个地方,他只是看不见。让一个海外工程师在几周内独立搞定一个多卡 OOM,本身就不太现实。但今天有了知乎这样的资源,他在中文搜索里点几下,几分钟就能看到一个 GPU 配置类似、踩过同一个坑的工程师写的回答。

然后是 **GitHub Issue** 这件事。过去开源模型的用户,得用英文写 Issue,等回复,跟 maintainer 的时区对接,各种琐碎的事情自己来。现在很多中国开源项目在 GitHub 上运营,海外用户可以直接用中文写 Issue,把回答、调试、验证这些事 outsource 给 maintainer。当然,要拿到真正的回复,他自己得能用中文清楚地表达问题。这就是这个课程在做的事情之一。

最后是 **汉字这个东西本身**。我觉得这是最有意思的一点。

对中文母语者来说,汉字是空气,你天天用,不会觉得它有什么特别。但对外国学习者来说,汉字是一种 "意义的积木" —— 每学会一个汉字,它会同时点亮几十个相关词。学会 "推",马上就能更快读懂 "推理"、"推论"、"推断"。学会 "量",量化、增量、考量都跟着进来。换句话说,汉字不是一个一个学单词,是学了一块积木,然后这块积木可以拼进几百个词。

这件事对韩国人和日本人来说,起跑线本来就高 —— 因为这两种语言里有大量从汉字来的词。一个韩国开发者第一次看到 「部署」,凭韩语汉字音的直觉就能猜出大概是 deploy,这是他从小学语言时已经积累的资本。但即便是没有这个资本的英文学习者,在掌握了 500-800 个核心汉字之后,也能用积木式的方式快速覆盖 MLOps 领域的几千个词。

所以这个课程的策略,不是从日常会话开始,而是从 MLOps 领域出现的汉字优先开始学。这个思路在几年前还是个别自学者私下用的小聪明,今天它已经变成了海外开发者快速读懂中文一手资料的最重要策略。

我刚才讲的这一切,只是想告诉大家一件事:**今天是开始读中文的一个好时机** —— 至少对外国开发者来说是。

换个话题。

这是一张图。图上的数字是 100、5、1。这几个数字代表什么?光看猜不出来。提示一下:最上面的 100,是五年前 LLM 一手资料里英文的占比;最下面的 1,是同一时期韩文的占比;中间的 5 是中文。

那如果把同一张图重画一遍,以 2025 年为基准呢?英文还是 100。但中文从 5 跳到了 60、70 这个量级。韩文还是在 1 附近。英文资料没有变少,是中文资料爆发了。

DeepSeek 用 R1 让全世界为之侧目。团队规模不大,模型开源,全球用户上千万。但运营这个模型所需要的一手信息,有相当一部分藏在中文的内部 wiki 和 model card 里。被翻译成英文的部分,可能就是一两页。而这一两页的英文版,可能在发布当天信息量是足的,但很可能没有像中文原版那样持续更新。

也就是说,一个被全球上千万人使用的推理基础设施,90% 以上的一手文档只存在于中文。这件事五年前是想都不敢想的。

实现这个变化的,正是我前面说的那些东西 —— ModelScope、知乎、中文 GitHub Issue、汉字的积木结构。这几个力叠在一起,造就了一手资料获取方式的剧烈变化。抓住这个机会的,是那些把中文当成一手语言来读的工程师、研究者和创业者 —— 不只在中国,也包括开始学中文的海外开发者。

你们再仔细看这张图。底下的资料层,是给上面的资料层提供一手信息的。上面的英文媒体,在引用底下的中文一手资料。想想看,arXiv 是个老牌、受人尊敬的平台,知乎是个活跃、热闹的平台。哪个更 "有名"?也许是 arXiv。但中国大模型公司的工程师,每天都在刷知乎。尤其是 arXiv 不提供 troubleshooting 信息,这一点更明显。在真实的生产环境调试场景里,知乎可能比 arXiv 还更有用。

时代变了,主角也跟着变。我做这张图的时候,自己都觉得结论挺震撼的。

它让我想到另一件事 —— 我们生活的这个**信息生态系统**。过去,主导它的语言基本就是英文。今天,英文和中文两条主线开始并立。**双语一手内容的时代到来了**。

最典型的例子就是 DeepSeek。推理模型的新王者,英文论文当然出,但真正的细节都在中文博客和 ModelScope 页面上。短短几个月,刷遍了全球头条 —— 你们都看到了。

还有另一群人,我也想提一下。看这个,**自动翻译 AI 新闻**。我每天看英文媒体,每次刷,自动翻译稿件就多一格。然后我就想 —— 那些被翻译扔掉的细节,都去哪儿了?会怎么样?

这样一看,自动翻译这片空间,其实是大多数人的便利和少数人的精确性之间的置换。这种现象只发生在自动翻译里吗?DeepSeek 团队当然聪明。但自动翻译引擎也是各自岗位上很能干的工具。即便如此,信息还是在丢、细节还是在被边缘化,原因是什么?是 **语言**。

信息是关于一手出处的问题。今天最强的一手出处,是它原本被写出来的那种语言。所以今天最重要的信息权力,我认为是 **用原文阅读的能力**。

矛(攻击),它的对面是什么?盾(防御)。矛和盾。这是矛盾这个词的本意。

如果你想做出一番事业、积累影响力,你必须懂一手资料。如果你想保护自己、保护你的团队、保护那些你在乎的人,不被别人手里那把信息差的剑伤到,你也必须懂一手资料。这听起来矛盾,但你们都知道,现实就是在这种矛盾之间运行的。

那应该学哪种语言?这个世界上语言很多。但语言之间不是平等的,在信息层级上有顺序。能控制信息的语言现在是英文和中文。对一个不会中文的海外工程师来说,**性价比最高的信息接入语言,就是中文**。换种说法,就是汉字阅读能力。

所以这个课程想告诉海外学员的一句话是:**今天是该读中文的一天**。

那我现在反过来,把这段话讲给屏幕前的中文母语朋友。

你们已经站在这个语言的中心,默认就在读一手资料。我不需要劝你们学中文。但我想请你们注意三件事。

**第一件事**,你写的中文,正在被海外的开发者认真地读。当你在知乎写一篇 vLLM 的踩坑回答,当你在 CSDN 写一篇量化部署的实操记录,当你在掘金写一篇微调心得 —— 这些内容现在不只是给国内同行看的了。它们是全球这一代 LLM 工程师能不能搞清楚某个细节的关键。请把这件事记在心里。

**第二件事**,如果你看到一个海外开发者用磕磕巴巴的中文给你的项目提了一个 GitHub Issue,或者在你的知乎回答下面用奇怪的语序留了评论,请对他温柔一点。他可能正在用四个月的时间,认真地学你写的这种语言,只是为了能直接读你的代码和你的文字。在他那一边,这是一笔不小的投资。

**第三件事**,如果你自己反过来想读懂英文圈的一手资料,我前面讲的那一套学习策略,反过来也成立。**不要从日常会话起步**。直接进入领域词汇,直接去读 arXiv、Hacker News、ICML 论文。把英文也当成一种 "汉字以外的积木系统" 去拆解 —— 词根、前缀、后缀,这些就是英文的偏旁部首。MLOps 领域核心英文术语其实只有几百个,跟我们让海外学员先攻的核心汉字数量,是同一个量级。

回到一开始那个问题 —— 那些只读不说的外语班都去哪儿了?会话班遍地都是,但教你 "读懂技术文档而不需要被声调和对话练习耗尽" 的地方,几乎找不到。有句俗话叫,**狗屎到了真要用作药引子的那一刻,反而找不到了**。所以我们就开了这么一个小小的、有点土气的项目 —— MLOps Language Curriculum。

谢谢大家。

## 给中文母语朋友的额外说明

如果你是中文母语者,读到这里,我想直接告诉你三件实用的事:

1. **你不是这个课程的学员,你是这个课程的资源**。课程里所有真实的学习材料(模型卡、知乎答案、CSDN 文章)都来自你和你的同行。你的写作就是教学材料。

2. **如果你想反向学习英文,这套策略可以照搬**。把 "先学领域汉字" 换成 "先学领域词根"。把 ModelScope 换成 Hugging Face。把知乎换成 Hacker News 或 lobste.rs。结构不变。

3. **欢迎用 PR 帮我们查中文表述**。如果你看到课程里某句中文翻译生硬、术语不地道,提个 PR,我们都接得住。中国 OSS 的礼貌惯例会真的会教外国学员 "原来用中文写 issue 应该是这种语气" —— 这件事比你想象的影响大。
