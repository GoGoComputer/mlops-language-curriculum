# 学习指南 02 — 海外学员的词汇精通(从入门到 220 词毕业)

> 第二讲(视频)的延伸阅读。给中文母语者看的版本。
> 假设你已经看过 [guide-00 零基础入门](guide-00-foundations.md) —— 那一份介绍了海外学员怎么处理汉字、部首、拼音、声调。
> 这一份在那个基础上,告诉你 「他们到底怎么把 220 个领域词内化的」,以及你写中文时哪些细节会影响他们的学习速度。
> 通读约 30 分钟。

## 一、海外学员要解决的核心难题:为什么不从 「你好」 起步

中文母语者从来不会问 「为什么先学这个词,不先学那个词」 —— 因为母语习得是从 「你好」 「妈妈」 「吃饭」 自然开始的。

但海外学员的目标完全不同。他们要在 4 个月内读懂 ModelScope 的模型卡,而不是去中国旅游。这一前提改变了一切。

我做了一个统计。把 ModelScope 上一篇 Qwen3 模型卡的前 1000 字,跟 HSK 各级词汇库做交集:

```
HSK 1 级 (150 个词)         → 模型卡里出现约 25 个
HSK 2 级 (累计 300 个)      → 50 个
HSK 3 级 (累计 600 个)      → 80 个
HSK 4 级 (累计 1200 个)     → 120 个

模型卡里出现、HSK 任何级别都没有的领域词:
                          → 200 个以上
```

也就是说,海外学员就算把 HSK 1 级背完,也只能覆盖一篇模型卡的 12%。即便背到 HSK 4 级(1200 个),也只能到 60-70%。剩下的 30-40% 全是技术领域词,通用课程根本不教。

所以我们让海外学员**跳过 HSK,直接攻 220 个 MLOps 领域词**。这是基于一个关键事实:**领域词的频率分布比日常词更陡峭**。在日常中文里,1500-2000 个词覆盖 80% 的对话;在 MLOps 中文里,200-300 个词覆盖 80% 的模型卡。原因是领域词重复得离谱 —— 「模型」「训练」「推理」「部署」「量化」 这五个词在每一篇模型卡里都会出现多次。

中文母语者读到这里大概会觉得 「废话,这些词当然到处都是」。但对海外学员来说,这个 「重复」 才是他们 4 个月毕业的真正原因。一个领域,一个频率分布,一份精确的 220 词清单 —— 这就是他们的入场券。

## 二、海外学员的核心策略:词为单位,字为副产物

海外学员经常会问一个问题:**「我应该先学汉字还是先学词?」** 这个问题在中文母语者听来很奇怪,因为对你而言,词就是词,你不会觉得 「先记『部』再记『部署』」 是个选项。

但对海外学员,这是一个真实的学习路径选择。我们的答案是:**记词,字是副产物**。

为什么?因为汉字会自动重复。比如 「模」 这个字,会出现在:

```
模型 (model)
模板 (template)
模式 (pattern, mode)
规模 (scale)
```

学员把这 4 个词放进 Anki,「模」 这个字就在他眼前出现了 4 次。他不需要专门去 「学汉字」,因为字会通过词反复登场。这是为什么我们让他们直接背 220 个词,而不是先背 1500 个汉字。

### 但他们还是受益于 「字的积木效应」

虽然学习单位是词,海外学员会通过词学到字,然后字会反过来帮助他们读新词。比如学了 「推」(出现在 推理 中),后面遇到:

```
推论 → 推 + 论(理论)→ 推论 = deduction
推断 → 推 + 断(判断)→ 推断 = inference (不同含义)
推荐 → 推 + 荐(举荐)→ 推荐 = recommendation
推动 → 推 + 动(动作)→ 推动 = drive forward
```

学员在 Pleco 里查 「推动」 时,看到 「推」 就秒懂半个意思,只需要再认识 「动」 即可。这个 「积木重组」 是他们超过 200 词后的核心加速机制。

中文母语者你大概不会意识到这件事 —— 因为你从小就懂 「推」 这个字,从没想过它是 「积木」。但对学员来说,这是他们最依赖的工具。

## 三、220 个词的分布与中文母语者的关系

我们的词汇主文件 `vocab/mlops-domain-vocab.csv` 把 220 词分成五个类别:

| 类别 | 词数 | 中文母语者读起来的感觉 |
|------|------|----------------------|
| 模型与架构 | 70+ | 工作日常用语 |
| 基础设施与部署 | 50+ | 工作日常用语 |
| 性能与评估 | 20+ | 测评报告用语 |
| 数据与管道 | 25+ | 工作日常用语 |
| 通用动词与连接词 | 50+ | 写技术文档时下意识使用 |

每一个词对你来说都是 「啊就是那个意思嘛」,但对学员来说,每一个都是要刻意压进脑子的拼图。

### 给中文母语朋友的有趣观察

我们让海外学员先攻第五类(通用动词与连接词),然后第一类(模型与架构),再依次往后。原因是,**没有第五类,句子结构整个解不开**。

「支持」「采用」「基于」「通过」「用于」「需要」 这六个词,中文母语者写技术文时连思考都不需要。但对学员来说,这六个词是 15 个核心句型的载体 —— 学到这六个词的同时,他也学到了对应的句型。一举两得。

所以我们让学员第一周就攻这六个,然后才开始 「模型」 「训练」 这种领域专有名词。优先级跟传统课程完全反过来。

## 四、海外学员的 Anki 工作流(4 步)

为了让你了解他们的具体节奏:

```
Step 1: 跑构建脚本
       python3 scripts/build_vocab.py
       生成 vocab/build/vocab-{ko,en,zh}.csv

Step 2: 装 Anki(免费桌面端 + 付费手机端)

Step 3: 导入 CSV
       File → Import → vocab-{lang}.csv
       Front: 汉字  Back: 母语翻译 + 拼音

Step 4: 每天 30 张新卡,无限复习
       韩语学员一个月内完成 220 张
       英语学员可能 1.5 个月,因为没有汉字直觉
```

每天 10 分钟通勤路上刷,加上晚上 25 分钟读真实材料,这就是他们 Month 1 的全部日常。

## 五、海外学员的母语优势对照

### 韩语母语者:汉字音直接映射

韩语里 70% 是汉字词,所以这部分学员看到 「部署」 就直接对应到韩语 「부서」,意思 90% 命中。这是他们小学到大默默积累的资本。

| 中文 | 韩语汉字音 | 意思 |
|------|----------|------|
| 模型 | 모형 | model ✓ |
| 训练 | 훈련 | training ✓ |
| 部署 | 부서 | deployment ✓ |
| 量化 | 양화 | quantization ✓ |
| 微调 | 미조 | fine-tuning ✓ |

### 英语母语者:汉字拆解积木

没有韩语学员的捷径,他们用的是字的积木:

```
学一个 「推」(push, derive)
  → 推理 / 推论 / 推断 / 推荐 / 推动 全部点亮

学一个 「量」(measure)
  → 量化 / 增量 / 考量 / 测量 全部点亮
```

英语学员需要刻意学约 500 个汉字,才能达到韩语学员凭直觉就有的覆盖率。所以英语学员需要 6-8 个月,韩语学员 4 个月。

## 六、给中文母语朋友的三件事

### 第一件:你的写作是他们的教材

学员不读传统教材。他们读 你 写的模型卡、 你 写的知乎答案、 你 写的 CSDN 博客。所以请你:

**(a) 段落短一点。** 学员一段读 5-10 分钟,过长的段他读不完。一段写到 5-7 句以内最理想。

**(b) 术语稳定。** 同一篇里如果用了 「推理」,后面就一直 「推理」,不要突然切到 「inference」 或 「推断」。中文读者无所谓,但学员的词汇匹配会断三次,每次都要查字典。

**(c) 用核心 15 句型,不要太花哨。** 直接用 「基于...架构」「用于...」「采用...设计」「对...进行...」 这种结构,不要写成 「这模型骨子里其实就是 Transformer」 之类口语化或文学化的表达。技术文档不是博客,清晰的中文是面向全球的最佳中文。

### 第二件:见到磕磕巴巴的中文请温柔一点

如果你在 Issue 评论区或者知乎答案下看到一个外国人用中文留言,语气有点 「太规整」「过于书面」 —— 比如他每句话都是 「该模型...」「本工具...」「对 X 进行 Y」 —— 不要笑。那是他刚学完 15 个核心句型,把它们全部用上的样子。一年后他自然会松弛下来。这跟我们刚学英文时只会用一两个固定句型,是一个道理。

### 第三件:中文母语者怎么用同样的方法读懂英文一手资料

如果你想反过来快速读 PyTorch / vLLM / Hacker News:

**(a) 不要从口语词起步。** "don't, gonna, hi" 这些跟你想读的英文 README 完全不重叠。

**(b) 直接攻领域词根。** 英文里有词根:

```
infer-     → inference, inferring, inferred, inferential
deploy-    → deployment, deploying, redeployment
quant-     → quantization, quantize, dequantize
align-     → alignment, aligning, misalignment
optim-     → optimize, optimization, optimal
```

学 50 个 MLOps 词根 ≈ 学 500 个英文技术词。跟我们让海外学员先攻 500 个汉字解锁数千个中文词,是同一个机制。

**(c) 直接攻技术写作语法。** 英文 README 的语法骨架其实只有 12-15 个固定句型(passive、it is...that、given that、provided that、in order to、by means of、对应中文的 「被」「是...的」「如果...就」「通过」),不需要学 「How are you」「Nice to meet you」 那一套。

把 ModelScope 换成 Hugging Face,把知乎换成 Hacker News 或 lobste.rs。4 个月路线图照搬过去,一样能毕业。
