# 학습 가이드 04 — 모델 카드 독해 마스터 (실전 5단락 졸업)

> 강의 04(영상)의 보충 학습 자료입니다.
> [0단계 가이드](guide-00-foundations.md), [어휘 마스터(02)](guide-02-vocabulary.md), [문법 마스터(03)](guide-03-grammar.md)를 모두 통독하셨다고 가정합니다.
> 이 가이드는 그 모든 학습이 합쳐져서 실제 모델 카드를 읽을 때 어떻게 작동하는지를 자세히 시연하는 자료입니다.
> 통독 약 60분. 그 후 모델 카드 한 편 읽을 때마다 다시 펴보시기를 권장합니다.

---

## 0. 이 가이드의 위치

이 시점에서 여러분은 다음을 갖추셨다고 가정합니다.

- **0단계 통과**: 한자가 무엇인지, 부수가 어떻게 작동하는지, 병음과 사성을 어떻게 다루는지 직관 보유
- **220개 도메인 어휘**: Anki에서 80% 적중률로 외워진 상태
- **15개 핵심 문형**: 자료에서 만났을 때 식별 가능한 상태

이 위에서 본격적인 모델 카드 독해를 시작합니다. 이 가이드를 다 보시면 모델 카드 5편을 자기 힘으로 풀어내실 수 있게 돼요. 그게 졸업 조건입니다.

---

## 1. 모델 카드란 무엇인가 — 공통 구조

모델 카드는 **ML 모델의 표준 문서**입니다. ModelScope, Hugging Face, ms-swift, FastChat 등 어디서든 거의 같은 구조를 가져요.

이 표준 구조를 알고 있는 게 독해 효율의 첫 번째 가속기에요. **어디에 어떤 정보가 있는지 미리 알면 검색 시간이 폭발적으로 줄어듭니다.**

### 5개 섹션의 표준 구조

거의 모든 모델 카드는 다음 5개 섹션으로 구성됩니다:

| # | 섹션 | 중국어 표준 헤더 | 핵심 정보 |
|---|------|-----------------|-----------|
| 1 | 모델 소개 | `模型简介` / `简介` / `模型介绍` | 무엇 / 어떤 아키텍처 / 어디 사용 |
| 2 | 사양 | `模型规格` / `参数说明` / `模型参数` | 파라미터 수 / 컨텍스트 / 양자화 옵션 |
| 3 | 사용법 | `使用方法` / `快速开始` / `Quick Start` | Python 코드 + 환경 요구사항 |
| 4 | 벤치마크 | `评测结果` / `性能` / `基准测试` | 테이블 |
| 5 | 라이선스 | `许可证` / `使用协议` / `License` | 라이선스 + 인용 |

물론 추가 섹션이 있을 수도 있어요 (`使用案例` 사용 사례, `已知限制` 알려진 제한, `更新日志` 변경 이력 등). 하지만 위 5개는 거의 항상 있고, 이 5개를 알면 전체 90%는 위치 추적 가능합니다.

### 섹션별 우선순위

여러분의 학습 시간이 한정되어 있다면, 다음 순서로 우선순위를 두세요.

1. **1순위 — 모델 소개**: 가장 많이 읽게 됩니다. 정형구 패턴이 가장 명확.
2. **2순위 — 사양**: 숫자 + 한자 단위. 빠르게 익숙해집니다.
3. **3순위 — 벤치마크**: 표 위주. 헤더 어휘만 알면 됨.
4. **4순위 — 사용법**: 코드 블록은 영어. 한자는 보조 설명만.
5. **5순위 — 라이선스**: 짧고 정형. 쉽게 마스터.

처음에는 모델 소개와 사양에 집중하세요. 이 두 섹션이 모델 카드 독해의 70%를 차지합니다.

## 2. 섹션별 정형구 — 외워두면 90% 자동 인식

각 섹션마다 거의 항상 등장하는 정형구가 있어요. 이걸 외워두면 (외운다기보다 자료에서 5-10번 만나는 동안 자연스럽게 익숙해지면) 모델 카드 독해 속도가 폭발적으로 빨라집니다.

### 2.1 모델 소개 섹션의 정형구

```
{모델명} 是基于 {아키텍처} 的 {모델 종류},支持 {기능 1}、{기능 2} 等任务。
该模型采用 {핵심 기법} 设计,可用于 {용도 1}、{용도 2} 等场景。
```

**해석**:
- "{모델명}은 {아키텍처} 기반의 {모델 종류}이며, {기능 1}, {기능 2} 등의 작업을 지원한다."
- "이 모델은 {핵심 기법} 설계를 채택했으며, {용도 1}, {용도 2} 등의 시나리오에 사용될 수 있다."

**실제 예**:
- `Qwen3 是基于 Transformer 的大语言模型,支持文本生成、推理、对话等任务。`
- `该模型采用 MoE 设计,可用于多种自然语言处理场景。`

이 두 문장에서 강의 03에서 다룬 핵심 5문형 중 3개(基于, 用于, 是...的)가 모두 등장해요. 한 단락 분석으로 5문형 학습이 즉시 검증되는 셈.

### 2.2 사양 섹션의 정형구

```
总参数量为 {N} 亿
激活参数为 {M} 亿
支持上下文长度 {K}
最低显存要求 {V} GB
推荐显存 {W} GB
```

**해석**:
- "총 매개변수 수는 {N}억"
- "활성 매개변수는 {M}억" (MoE 모델일 때)
- "컨텍스트 길이 {K} 지원"
- "최소 VRAM 요구사항 {V}GB"
- "권장 VRAM {W}GB"

전부 정형구. 변수만 바뀌어요. 5번 읽으면 자동.

**숫자 단위 보충**:
- `亿` = 1억 (영어 100M)
- `万` = 1만 (영어 10K)
- `K` = 1천 (`千`도 같이 쓰임)
- `B` = 1B = `十亿` (영어 표기 그대로 자주 사용)

`235亿`는 23.5B (235M이 아니에요!), `1万亿`는 1T. 한국 한자음 그대로 "이백삼십오억"이라고 읽으시면 한국어 직관과 일치.

### 2.3 사용법 섹션의 정형구

```
首先,安装依赖:    (먼저, 의존성 설치)
然后,加载模型:    (그 다음, 모델 로딩)
最后,运行推理:    (마지막, 추론 실행)
```

또는:

```
按照以下步骤使用:    (다음 단계대로 사용)
1. 安装 transformers
2. 下载模型权重
3. 加载模型
4. 运行推理
```

코드 블록은 영어 기반이라 그대로 두시면 됩니다. 코드 사이에 끼는 한자 주석만 식별하면 OK. 이 섹션에서 가장 자주 등장하는 한자:

```
首先 (먼저)
然后 (그 다음)
最后 (마지막)
注意 (주의)
确保 (확인)
推荐 (추천)
建议 (권장)
```

### 2.4 벤치마크 섹션 — 표 헤더 어휘

벤치마크 섹션은 표 위주라 헤더 어휘만 알면 충분.

| 중국어 헤더 | 의미 |
|------------|------|
| 模型 / Model | 모델 이름 |
| 评分 / Score | 점수 |
| 准确率 | 정확도 (accuracy) |
| 召回率 | 재현율 (recall) |
| F1 / F1 分数 | F1 점수 |
| 吞吐量 | throughput |
| 时延 / 延迟 | latency |
| 速度 | 속도 |
| 对比 / 比较 | 비교 |
| 提升 | 향상 |
| 下降 | 하락 |
| 平均 | 평균 |

자주 비교 대상으로 등장하는 모델 이름들 (한자로):

```
GPT-4 / Claude / Gemini  — 영어 그대로
通义千问 (Qwen)          — 중국어 명칭
深度求索 (DeepSeek)      — 중국어 명칭
智谱 / GLM (ChatGLM)     — 중국어 명칭
零一万物 (01.AI / Yi)    — 중국어 명칭
```

회사 이름이 한자로 등장하는 경우가 있어요. 한 번 익숙해지면 끝.

### 2.5 라이선스 섹션의 정형구

```
本模型采用 {라이선스 이름} 协议发布。
商用 {允许/禁止/需要授权}。
学术使用 {允许}。
```

**해석**:
- "본 모델은 {라이선스} 협정에 따라 발표된다"
- "상용 {허용/금지/허가 필요}"
- "학술 사용 {허용}"

대표 라이선스 이름:

```
Apache 2.0    — 가장 자주
MIT           — 자주
GPL / LGPL    — 가끔
专属许可证 (전용 라이선스) — 회사 자체 라이선스 (주의)
```

이 섹션은 5분이면 마스터.

## 3. 실전 독해 5단락 — 단계별 분석 시연

이제 실제 모델 카드 단락 5개를 한 단락씩 자세히 풀어드립니다. 각 단락은 실제 모델 카드들의 일반적 표현 조합으로 만든 학습용 가상 텍스트예요.

각 단락마다:
1. 원문
2. 단어 단위 분해
3. 문형 식별
4. 통합 번역
5. 학습 포인트

### 3.1 Qwen 시리즈 스타일

**원문**:

```
Qwen3-235B-A22B 是基于 Transformer 架构的大语言模型,
支持文本生成、对话、推理等任务。
该模型采用 MoE 设计,总参数量为 2350 亿,激活参数为 220 亿,
可用于多种自然语言处理场景。
```

**Step 1 — 단어 분해**:

| 단어 | 한국 한자음 | 의미 |
|------|-----------|------|
| 是 | 시 | ~이다 |
| 基于 | 기우 | 기반으로 |
| 架构 | 가구 | 아키텍처 |
| 的 | (조사) | ~의/하는 |
| 大语言模型 | 대언어모형 | 거대 언어 모델, LLM |
| 支持 | 지지 | 지원하다 |
| 文本生成 | 문본생성 | 텍스트 생성 |
| 对话 | 대화 | 대화 |
| 推理 | 추리 | 추론, inference |
| 等 | 등 | 등 |
| 任务 | 임무 | 작업, 태스크 |
| 该模型 | 해당 모형 | 해당 모델, 이 모델 |
| 采用 | 채용 | 채택하다 |
| 设计 | 설계 | 설계 |
| 总参数量 | 총참수량 | 총 매개변수 수 |
| 为 | 위 | ~이다 |
| 亿 | 억 | 억 (100M) |
| 激活参数 | 격활참수 | 활성 매개변수 |
| 可用于 | 가용우 | 사용 가능 |
| 多种 | 다종 | 다양한 |
| 自然语言处理 | 자연언어처리 | NLP |
| 场景 | 장경 | 시나리오/사용 케이스 |

**Step 2 — 문형 식별**:

| 부분 | 문형 |
|------|------|
| `基于 Transformer 架构` | 3.4 基于 |
| `支持...等任务` | 4.7 支持 + 等 정형 |
| `采用 MoE 设计` | 기본 동사+목적어 |
| `总参数量为 2350 亿` | 정형구 (사양 섹션) |
| `可用于...场景` | 3.3 用于 (可+用于) |

**Step 3 — 통합 번역**:

> Qwen3-235B-A22B는 Transformer 아키텍처 기반의 거대 언어 모델로, 텍스트 생성, 대화, 추론 등의 작업을 지원한다. 이 모델은 MoE 설계를 채택했으며, 총 매개변수 수는 2350억, 활성 매개변수는 220억으로, 다양한 자연어 처리 시나리오에 사용될 수 있다.

**Step 4 — 학습 포인트**:

- `2350 亿` = 235B (한국 한자음 "이천삼백오십억")
- `MoE` = Mixture of Experts. 영어 약어 그대로 등장. 총 vs 활성 파라미터 차이가 MoE의 특징.
- `自然语言处理` = NLP. 한국 한자음 그대로 "자연언어처리".

### 3.2 DeepSeek 시리즈 스타일

**원문**:

```
DeepSeek-R1 是一款专注于推理任务的大模型,
通过强化学习对模型进行对齐训练。
相比之前的版本,本次发布在数学推理和代码生成方面性能显著提升。
```

**Step 1 — 단어 분해**:

| 단어 | 한국 한자음 | 의미 |
|------|-----------|------|
| 一款 | 일관 | 한 종류의 (양사) |
| 专注于 | 전주우 | ~에 전념하다, 특화하다 |
| 推理任务 | 추리임무 | 추론 작업 |
| 通过 | 통과 | ~를 통해 |
| 强化学习 | 강화학습 | 강화학습, RL |
| 对齐训练 | 대제훈련 | 정렬 훈련, alignment training |
| 相比 | 상비 | ~와 비교하여 |
| 之前的版本 | 지전적판본 | 이전 버전 |
| 本次发布 | 본차발포 | 본 릴리스 |
| 在...方面 | 재...방면 | ~측면에서 |
| 数学推理 | 수학추리 | 수학 추론 |
| 代码生成 | 대마생성 | 코드 생성 |
| 性能 | 성능 | 성능 |
| 显著提升 | 현저제승 | 현저히 향상 |

**Step 2 — 문형 식별**:

| 부분 | 문형 |
|------|------|
| `是一款...的` | 4.1 是...的 (강조) |
| `专注于...` | 用于 변형 (전용 의미 강화) |
| `通过 X 对 Y 进行 Z` | 4.5 通过 + 3.5 对...进行 (복합) |
| `相比之前的版本` | 비교 표현 |
| `在...方面` | 영역 표현 |

**Step 3 — 통합 번역**:

> DeepSeek-R1은 추론 작업에 특화된 거대 모델로, 강화학습을 통해 모델에 대해 정렬 훈련을 수행한다. 이전 버전과 비교하여 본 릴리스는 수학 추론 및 코드 생성 측면에서 성능이 현저히 향상되었다.

**Step 4 — 학습 포인트**:

- `一款` = 양사. 모델, 제품, 게임 등의 단위. 영어 "a" 또는 "one (kind of)" 정도. 한국어로는 옮길 필요 없음.
- `通过 X 对 Y 进行 Z` 복합 구조 = "X를 통해 Y에 대해 Z를 수행" = "by X, perform Z on Y". 모델 카드의 핵심 빈출 패턴.
- `显著提升` = "현저히 향상". `显著`은 한국어 한자음 "현저" 그대로.

### 3.3 ChatGLM 스타일

**원문**:

```
若您的显存不足 16GB,建议使用 INT4 量化版本。
具体使用方法如下:首先安装 transformers,然后从 ModelScope 下载模型权重。
```

**Step 1 — 단어 분해**:

| 단어 | 한국 한자음 | 의미 |
|------|-----------|------|
| 若 | 약 | 만약 (如果의 격식체) |
| 您 | 닌 | 당신 (你의 격식체) |
| 显存 | 현존 | VRAM |
| 不足 | 부족 | 부족하다 |
| 建议 | 건의 | 권장하다 |
| 使用 | 사용 | 사용하다 |
| 量化版本 | 양화판본 | 양자화 버전 |
| 具体 | 구체 | 구체적 |
| 使用方法 | 사용방법 | 사용법 |
| 如下 | 여하 | 다음과 같이 |
| 首先 | 수선 | 먼저 |
| 安装 | 안장 | 설치 |
| 然后 | 연후 | 그 다음 |
| 从 | 종 | ~에서 |
| 下载 | 하재 | 다운로드 |
| 模型权重 | 모형권중 | 모델 가중치 |

**Step 2 — 문형 식별**:

| 부분 | 문형 |
|------|------|
| `若...,建议...` | 4.4 如果...就 변형 (若 + 建议) |
| `不足 16GB` | 부정 표현 (부족 + 수치) |
| `具体...如下` | 정형구 (사용법 도입) |
| `首先...然后...` | 순서 표현 |

**Step 3 — 통합 번역**:

> VRAM이 16GB 미만이라면, INT4 양자화 버전 사용을 권장합니다. 구체적인 사용법은 다음과 같습니다: 먼저 transformers를 설치하고, 그 다음 ModelScope에서 모델 가중치를 다운로드하세요.

**Step 4 — 학습 포인트**:

- `若` 같은 격식체 어휘는 모델 카드 / 공식 문서에 자주 등장. `如果`보다 더 격식 있음.
- `您` = `你`의 존칭. 사용자 매뉴얼/안내문에 자주.
- `首先...然后...` 순서 표현은 사용법 섹션의 표준. 5번 보면 자동.

---

### 3.4 Yi 시리즈 스타일

**원문**:

```
本模型在 200K 长上下文下表现稳定,
适用于文档分析、长文本摘要等场景。
注意:推理时显存占用较高,建议使用 80GB 以上的 GPU。
```

**Step 1 — 단어 분해**:

| 단어 | 한국 한자음 | 의미 |
|------|-----------|------|
| 本模型 | 본모형 | 본 모델 |
| 在 | 재 | ~에서 |
| 长上下文 | 장상하문 | 장기 컨텍스트 |
| 下 | 하 | ~하에서 |
| 表现 | 표현 | (성능을) 보이다 |
| 稳定 | 온정 | 안정적 |
| 适用于 | 적용우 | ~에 적용 가능 |
| 文档分析 | 문당분석 | 문서 분석 |
| 长文本摘要 | 장문본적요 | 장문 요약 |
| 注意 | 주의 | 주의 |
| 推理时 | 추리시 | 추론 시 |
| 占用 | 점용 | 점유 |
| 较高 | 교고 | 비교적 높다 |
| 以上 | 이상 | 이상 |

**Step 2 — 문형 식별**:

| 부분 | 문형 |
|------|------|
| `在...下` | 조건 표현 (~하에서) |
| `适用于...等场景` | 3.3 用于 변형 + 等 정형 |
| `注意:` | 주의 헤더 |
| `较高` | 정도 표현 (-er) |
| `建议使用 80GB 以上的 GPU` | 4.6 需要 변형 (建议=권장) |

**Step 3 — 통합 번역**:

> 본 모델은 200K 장기 컨텍스트하에서 안정적인 성능을 보이며, 문서 분석, 장문 요약 등의 시나리오에 적용 가능합니다. 주의: 추론 시 VRAM 점유가 비교적 높으므로 80GB 이상의 GPU 사용을 권장합니다.

**Step 4 — 학습 포인트**:

- `在 X 下` = "X 조건하에서". 영어 "under X" / "in X conditions". 모델 카드에서 환경 명시할 때 자주.
- `较高 / 较低` = "비교적 높다/낮다". `较` = "비교적".
- `200K`처럼 영어 단위 그대로 쓰는 경우가 많아요. 한자와 영어 혼용은 정상.

### 3.5 MiniMax 스타일

**원문**:

```
通过对模型进行 INT4 量化,显存占用减少 75%,推理速度提升约 2 倍。
量化后的模型在常见基准测试上的性能下降控制在 1% 以内。
```

**Step 1 — 단어 분해**:

| 단어 | 한국 한자음 | 의미 |
|------|-----------|------|
| 通过 | 통과 | ~를 통해 |
| 对 | 대 | ~에 대해 |
| 进行 | 진행 | 수행하다 |
| INT4 量化 | INT4 양화 | INT4 양자화 |
| 减少 | 감소 | 감소 |
| 75% | 75% | 75퍼센트 |
| 推理速度 | 추리속도 | 추론 속도 |
| 提升 | 제승 | 향상 |
| 约 | 약 | 약 (대략) |
| 倍 | 배 | 배수 |
| 量化后 | 양화후 | 양자화 후 |
| 常见 | 상견 | 일반적인 |
| 基准测试 | 기준측시 | 벤치마크 테스트 |
| 性能下降 | 성능하강 | 성능 하락 |
| 控制 | 제어 | 제어/유지 |
| 以内 | 이내 | 이내 |

**Step 2 — 문형 식별**:

| 부분 | 문형 |
|------|------|
| `通过对模型进行 INT4 量化` | 4.5 通过 + 3.5 对...进行 (복합) |
| `减少 75%, 提升约 2 倍` | 정도/수치 표현 |
| `量化后的模型` | 시간 후 표현 (后의) |
| `在...上` | 영역 표현 |
| `控制在 X 以内` | 정형구 (X 이내로 제어) |

**Step 3 — 통합 번역**:

> 모델에 대해 INT4 양자화를 수행함으로써 VRAM 점유가 75% 감소하고 추론 속도가 약 2배 향상된다. 양자화된 모델의 일반적인 벤치마크 테스트에서의 성능 하락은 1% 이내로 제어된다.

**Step 4 — 학습 포인트**:

- `通过对X进行Y` = 모델 카드의 핵심 빈출 복합 패턴. 한 번 외워두면 자료 어디서든 인식.
- `控制在 X 以内` = "X 이내로 제어". 성능 손실 표현의 정형구.
- `约` = "대략" (한국 한자음 "약" 그대로). 수치 어림짐작 표현에 자주.

## 4. 막혔을 때 워크플로우 — 7단계

자료를 읽다 보면 분명히 막히는 부분이 나옵니다. 그때를 위한 표준 워크플로우.

```
한 문장 만남
    ↓
[1단계] 도메인 어휘 식별 (220개 안에 있는가?)
    ↓
[2단계] 한자어 직관으로 의미 추측 (한국 한자음)
    ↓
[3단계] 문법 패턴 식별 (15문형 중 어떤 것?)
    ↓
[4단계] 막힌 단어만 Pleco 검색
    ↓
[5단계] 통합 번역 시도
    ↓
[6단계] DeepL과 비교 (1주일에 1번이면 충분)
    ↓
[7단계] 차이가 큰 부분 → 학습 노트
```

### 각 단계 자세히

**1단계 — 도메인 어휘 식별**

문장에서 220개 어휘에 들어가는 단어를 먼저 찾으세요. 보통 한 문장에 5-10개 단어 중 3-5개는 도메인 어휘에요. 이걸 먼저 식별하면 뼈대가 잡힙니다.

**2단계 — 한자어 직관**

도메인 어휘에 안 들어가는 단어를 한국 한자음으로 추측. 예: `具体`(구체) = 한국어 "구체적인". `实际上`(실제상) = "실제로". 70%는 한자음으로 의미가 잡혀요.

**3단계 — 문법 패턴 식별**

15문형 중 어떤 것이 등장하는지 표시. 把/将, 用于, 基于, 对...进行가 보이면 절반 끝.

**4단계 — Pleco 검색**

여기까지 해도 모르는 단어만 사전 찾기. 보통 한 문장에 1-2개. 시간 낭비 없음.

**5단계 — 통합 번역 시도**

한국어로 자연스럽게 옮겨보기. 단어 단위 직역에 매달리지 마세요. 자연스러운 한국어가 더 정확.

**6단계 — DeepL 비교 (선택)**

매주 일요일 한 단락 정도만. 자기 번역과 DeepL 차이를 표로 비교.

**7단계 — 학습 노트**

차이가 큰 부분이 학습 포인트에요. 어휘 누락? 문법 오해? 의역 vs 직역? 다음 주 학습 계획에 반영.

### "모든 단어를 다 알 필요 없다" 원칙

이 원칙은 강의 02에서도 강조했지만 실전에서 가장 중요합니다. 한 문장에 모르는 단어가 5개 있어도, 그중 2-3개의 의미만 잡히면 문장 전체 의미가 80% 잡혀요. 나머지 2-3개는 컨텍스트로 추측 가능.

**잘못된 흐름**: 모든 모르는 단어를 사전 찾고, Anki에 추가하고, 다음 문장 못 읽음.

**올바른 흐름**: 80% 의미만 잡으면 다음 문장으로. 같은 단어 5번 이상 만나면 그때 외움.

## 5. DeepL과 자기 번역 비교 학습법

DeepL은 우리의 검증 도구이지 정답이 아닙니다. 하지만 학습 초반에는 매우 유용해요.

### 일주일에 한 번 워크플로우

```
1. 모델 카드 한 단락 선택 (50-100자)
2. 자기 번역 (시간 측정)
3. DeepL 번역
4. 두 번역을 나란히 비교
5. 차이 분류:
   - 단어 누락 → 어휘 보충
   - 문법 오해 → grammar.md 다시 읽기
   - 의역/직역 차이 → 무시 (DeepL이 더 자연스럽지만 의미는 같음)
6. 학습 노트에 기록
```

### 시기별 진척

**Week 1-4**: DeepL 압승. 자기 번역의 70% 정도 누락. 정상.

**Week 5-8**: 거의 비슷. 자기 번역의 90% 정도 정확. DeepL 가끔 더 자연스러운 한국어.

**Week 9-12**: 가끔 자기 번역이 더 정확한 순간. DeepL이 도메인 어휘를 잘못 번역하는 케이스 발견.

**Week 13-16**: 자기 번역이 신뢰 가능. DeepL은 보조용으로만.

이 진척을 직접 측정하시면, 슬럼프 때 동기부여가 됩니다.

## 6. 다음 단계 — Zhihu / CSDN로 진입

모델 카드 5편 졸업 후 다음 자료원:

| 자료원 | 난이도 | 특징 |
|--------|--------|------|
| ModelScope 모델 카드 | ★★ | 정형구 많음, 학습용 ideal |
| Hugging Face 中文 카드 | ★★★ | 영중 혼용, 약간 더 어려움 |
| Zhihu 트러블슈팅 | ★★★★ | 회화체 + 기술 혼합 |
| CSDN 기술 글 | ★★★★ | 비공식 톤, 도메인 깊이 |
| 掘金 (Juejin) 후기 | ★★★ | 짧고 구체적 |
| arXiv 中文 abstract | ★★★★★ | 학술 톤, 졸업 후 도전 |

**추천 진행 순서**: 모델 카드 5편 → Zhihu 답변 3편 → CSDN 글 2편 → arXiv abstract.

### 각 자료원 특징

**Zhihu**: 회화체와 기술 표현이 섞임. 어기 조사가 가끔 등장 (이때까지 무시했던 吧, 啊가 살짝 등장). 그래도 80%는 모델 카드와 같은 표현.

**CSDN**: 개인 블로그 톤. 비공식적이고 도메인 깊이가 더함. 모델 카드보다 어려운 어휘가 많이 등장. 학습 가치는 가장 큼.

**Juejin**: 짧고 실용적. 트러블슈팅 후기가 많음. 모델 카드와 Zhihu 사이의 난이도.

**arXiv**: 학술 톤, 격식 어휘 (将, 若, 以...为, 即 등) 많이 등장. 졸업 후에 도전.

## 7. GitHub Issue 작성 입문 — 졸업 산출물 1번

졸업 산출물 첫번째 — 실제 GitHub Issue 1건 게시.

### 표준 구조 (templates/github-issue-template.md 참고)

```markdown
### 问题描述 (Problem)
{한 문장으로 무슨 문제인지}

### 复现步骤 (Reproduction)
1. 我使用 ...
2. 然后 ...
3. 出现以下错误:
\`\`\`
{에러 로그}
\`\`\`

### 环境 (Environment)
- GPU: ...
- CUDA: ...
- PyTorch: ...
- 模型: ...

### 我尝试过的方法 (Attempted)
我尝试过 ...,但是 ...

### 期望行为 (Expected)
我希望 ...
```

각 섹션 1-2 문장이면 충분. **짧고 정확한 게 좋은 Issue.** 에러 로그는 영어 그대로 붙이면 됨.

### 자주 쓰는 표현 10개

```
我遇到了... — 나는 ~를 만났다 (문제 제보)
请问... — 여쭙겠습니다 (정중한 질문)
如何... — 어떻게 ~하나요?
是否支持... — ~를 지원하나요?
谢谢 — 감사합니다
请参考... — ~를 참고하세요
我已经尝试过... — 나는 이미 ~를 시도해봤다
但是没有解决 — 하지만 해결되지 않았다
期待您的回复 — 답변 기다리겠습니다
非常感谢 — 매우 감사합니다
```

이 10개로 거의 모든 GitHub Issue를 쓸 수 있어요.

### 타겟 OSS 프로젝트 추천

```
vLLM (PagedAttention 추론 엔진)
ms-swift (모델 학습/추론 통합)
FastChat (LLM 서빙)
ChatGLM (智谱 모델)
Qwen (Alibaba 모델)
DeepSeek-V3 (DeepSeek 모델)
LLaMA-Factory (학습 프레임워크)
```

본인의 실제 사용 경험이 있는 프로젝트를 고르시는 게 가장 좋아요. 가짜 이슈 만들지 마시고, 실제 막혔던 문제를 정리해서 게시.
