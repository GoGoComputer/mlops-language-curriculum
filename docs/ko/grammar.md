# 핵심 문법 15개 — 실전 예문집

> 실제 중국 기술 README / 논문 / 커뮤니티에서 자주 등장하는 문장 패턴 모음.
> 각 항목: **문형 → 의미 → 실전 예문 3개 → 한국어 직역 → 학습 포인트**

---

## 1. `是...的` — ~이다 (강조·확정 진술)

**의미**: 어떤 사실의 성격·시점·방식·주체를 강조. 기술 문서에서는 "이 모델은 ~로 만들어졌다" 식 진술의 핵심.

**예문**
- 这个模型**是**基于 Transformer 架构**的**。
  → 이 모델은 Transformer 아키텍처를 기반으로 한 것이다.
- 该项目**是**用 PyTorch 实现**的**。
  → 본 프로젝트는 PyTorch로 구현된 것이다.
- 这些参数**是**在预训练阶段确定**的**。
  → 이 파라미터들은 사전학습 단계에서 결정된 것이다.

**포인트**: `是 ... 的` 사이에 들어가는 게 강조 대상. 한국어 "~인 것이다"와 거의 1:1 대응.

---

## 2. `不 / 没` — 부정

**의미**: `不` = 일반적·미래 부정 / `没` = 발생하지 않음·소유하지 않음 (과거·완료 부정).

**예문**
- 当前版本**不**支持 Windows 系统。
  → 현재 버전은 Windows 시스템을 지원하지 않는다.
- 该参数**没**有默认值，必须显式指定。
  → 이 파라미터는 기본값이 없으며, 반드시 명시적으로 지정해야 한다.
- 我**没**遇到过这个错误。
  → 나는 이 에러를 만난 적이 없다.

**포인트**: "지원 안 함"은 거의 항상 `不支持`. 에러 보고할 때 "에러를 만났다"는 `遇到`, "안 만났다"는 `没遇到`.

---

## 3. `可以 / 能` — 가능

**의미**: `可以` = 허용·옵션 / `能` = 능력·물리적 가능. 기술 문서에서는 거의 호환 가능.

**예문**
- 你**可以**通过 API 调用该模型。
  → API를 통해 이 모델을 호출할 수 있다.
- 该模型**能**处理长达 128K 的上下文。
  → 이 모델은 최대 128K 길이의 컨텍스트를 처리할 수 있다.
- 用户**可以**自定义推理参数。
  → 사용자는 추론 파라미터를 커스텀할 수 있다.

**포인트**: "할 수 있다"의 표준 번역. README의 Features 섹션 거의 대부분이 이 구조.

---

## 4. `如果...就` — if ~ then

**의미**: 조건문. `就` 대신 `那么` `则`도 사용 (`则`은 문어체).

**예문**
- **如果**显存不足，**就**减小 batch size。
  → 만약 VRAM이 부족하면 batch size를 줄여라.
- **如果**模型加载失败，**则**检查权重文件路径。
  → 모델 로드가 실패하면 가중치 파일 경로를 확인하라.
- **如果**遇到 CUDA 错误，**那么**请提交 issue。
  → CUDA 에러가 발생하면 issue를 제출해주세요.

**포인트**: `则`이 나오면 공식 문서·논문 톤. README는 `就`, 학술 자료는 `则`.

---

## 5. `通过` — ~를 통해

**의미**: 수단·경로. 영어 "via / through / by"에 대응.

**예문**
- **通过** Docker 部署模型服务。
  → Docker를 통해 모델 서비스를 배포한다.
- **通过**调用 generate() 方法生成文本。
  → generate() 메서드를 호출함으로써 텍스트를 생성한다.
- 该指标**通过**人工评估得出。
  → 이 지표는 사람의 평가를 통해 도출된 것이다.

**포인트**: `通过 + 명사/동사구 + 동사`. 위치는 거의 항상 동사 앞.

---

## 6. `需要` — 필요하다

**의미**: 필요·요구. 시스템 요구사항·전제조건 진술의 핵심 어휘.

**예문**
- 推理至少**需要** 16GB 显存。
  → 추론에는 최소 16GB의 VRAM이 필요하다.
- 训练**需要**大量标注数据。
  → 학습에는 대량의 어노테이션 데이터가 필요하다.
- 您**需要**先安装 CUDA 12.1。
  → 먼저 CUDA 12.1을 설치할 필요가 있다.

**포인트**: System Requirements 섹션에서 거의 모든 줄에 등장.

---

## 7. `支持 / 不支持` — 지원·미지원

**의미**: 기능 지원 여부. README의 핵심 동사.

**예문**
- 本框架**支持**主流的深度学习模型。
  → 본 프레임워크는 주류 딥러닝 모델을 지원한다.
- 当前版本**不支持**多 GPU 训练，将在 v2.0 中支持。
  → 현재 버전은 멀티 GPU 학습을 지원하지 않으며, v2.0에서 지원할 예정이다.
- 该模型**支持**中英双语。
  → 이 모델은 중국어·영어 두 언어를 지원한다.

**포인트**: `支持` 뒤에 바로 명사가 옴. "곧 지원 예정"은 `即将支持`, "이미 지원"은 `已支持`.

---

## 8. `把` — 처분문 (X를 ~로 처리하다)

**의미**: 목적어를 동사 앞으로 끌어내 "그것을 어떻게 했는지" 강조. 코드 동작 설명에 빈출.

**예문**
- 请**把**模型转换为 ONNX 格式。
  → 모델을 ONNX 포맷으로 변환해주세요.
- 系统**把**输入数据切分成多个批次。
  → 시스템은 입력 데이터를 여러 배치로 분할한다.
- 我们**把**训练过程分为三个阶段。
  → 우리는 학습 과정을 세 단계로 나눈다.

**포인트**: `把 + 목적어 + 동사 + 결과`. "X를 Y로 변환·분할·처리하다" 패턴은 거의 다 `把` 사용.

---

## 9. `将` — 把의 문어체 (공식 문서 표준)

**의미**: 의미는 `把`와 같지만 문어체. 기술 문서·논문은 `将` 우선. 또한 미래 시제 "~할 것이다" 의미도 있음.

**예문**
- **将**输入张量传递给模型。
  → 입력 텐서를 모델에 전달한다.
- 我们**将**该方法与基线模型进行比较。
  → 우리는 이 방법을 베이스라인 모델과 비교한다.
- 下一版本**将**支持流式输出。
  → 다음 버전은 스트리밍 출력을 지원할 예정이다.

**포인트**: 문서 톤이 `将`이면 공식 톤, `把`면 친근한 톤. 두 의미(처분/미래)는 문맥으로 판별.

---

## 10. `被` — 피동

**의미**: ~당하다, ~되다. 에러 메시지·동작 결과 설명에 자주.

**예문**
- 该参数**被**忽略。
  → 이 파라미터는 무시된다.
- 模型**被**自动加载到 GPU。
  → 모델은 자동으로 GPU에 로드된다.
- 异常**被**捕获并记录到日志。
  → 예외가 캡처되어 로그에 기록된다.

**포인트**: 영어 수동태와 거의 동일. `被` 뒤에 행위자가 와도, 안 와도 됨.

---

## 11. `用于` — ~에 사용되다

**의미**: 용도·목적 진술. 모델 카드·논문 abstract의 단골.

**예문**
- 该模型**用于**自然语言处理任务。
  → 이 모델은 자연어 처리 태스크에 사용된다.
- 此参数**用于**控制生成长度。
  → 이 파라미터는 생성 길이를 제어하는 데 사용된다.
- 这些数据**用于**评估模型性能。
  → 이 데이터들은 모델 성능 평가에 사용된다.

**포인트**: `用于 + 목적/대상`. 모델 카드 첫 문장 거의 다 이 패턴.

---

## 12. `基于` — ~를 기반으로

**의미**: 기반·근거. README 첫 단락에 거의 무조건 등장.

**예문**
- 本模型**基于** Transformer 架构。
  → 본 모델은 Transformer 아키텍처를 기반으로 한다.
- 我们**基于**开源数据集进行训练。
  → 우리는 오픈소스 데이터셋을 기반으로 학습을 진행했다.
- 这是一个**基于** RAG 的问答系统。
  → 이것은 RAG 기반의 Q&A 시스템이다.

**포인트**: 영어 "based on"과 1:1. 명사 앞에서 수식어로도 자주 쓰임 (`基于 X 的 Y`).

---

## 13. `对...进行` — ~에 대해 ~를 수행하다

**의미**: 학술·기술 문어체의 정형구. "X에 대해 Y를 한다"의 무거운 버전.

**예문**
- **对**数据**进行**预处理。
  → 데이터에 대해 전처리를 수행한다.
- **对**模型**进行**微调。
  → 모델에 대해 파인튜닝을 진행한다.
- **对**实验结果**进行**统计分析。
  → 실험 결과에 대해 통계 분석을 수행한다.

**포인트**: 동사 하나로 끝낼 수 있는 걸 굳이 `对 + 명사 + 进行 + 동작명사`로 만든 표현. 논문에서 격식 톤을 만들기 위한 패턴. **이 구조 모르면 abstract 절반 못 읽음**.

---

## 14. `以...方式` / `以...为` — ~방식으로 / ~를 ~로 삼다

**의미**: 방식·수단·자격 진술. 기술 자료에서 격식 표현으로 자주.

**예문**
- **以**异步**方式**执行任务。
  → 비동기 방식으로 태스크를 실행한다.
- **以** JSON 格式返回结果。
  → JSON 포맷으로 결과를 반환한다.
- **以**准确率**为**主要评估指标。
  → 정확도를 주요 평가 지표로 삼는다.

**포인트**: `以 X 方式` = X 방식으로 / `以 X 为 Y` = X를 Y로 삼다. 후자는 학술 톤.

---

## 15. `得` — 정도·결과 보어

**의미**: 동사 뒤에 붙어서 "얼마나 ~하는지" 보충. 성능 비교·평가 진술의 핵심.

**예문**
- 该模型训练**得**很快。
  → 이 모델은 매우 빠르게 학습된다.
- 推理速度提升**得**非常明显。
  → 추론 속도가 매우 뚜렷하게 향상되었다.
- 这个方法效果**得**到了显著改善。
  → 이 방법의 효과가 현저히 개선되었다. (※ `得到` = 얻다, 별도 어휘)

**포인트**: 두 가지 헷갈리는 점.
- (a) `V + 得 + 정도부사 + 형용사`: "V를 얼마나 잘 하는지" (예: `跑得快` = 빠르게 달린다)
- (b) `得到`: 별개 단어, "얻다·받다" 의미.
이 둘 구분만 되면 끝.

## 추가 — Month 2 진입 전 알아둘 보너스 4개

### `所` + 동사 — 문어체 피동 명사화

- **所**支持的格式 → 지원되는 포맷
- **所**采用的方法 → 채택한 방법
- 模型**所**学到的特征 → 모델이 학습한 특징

→ 논문에서 명사구 만들 때 거의 필수.

### `即` — 즉, 다시 말해

- 这是一种零样本学习，**即**不需要任何训练样本。
  → 이것은 zero-shot 학습, 즉 어떠한 훈련 샘플도 필요로 하지 않는 학습이다.

→ 정의·재진술 도입. 영어 `i.e.` 와 동일.

### `如` — 예를 들어 / ~와 같은

- 主流框架**如** PyTorch、TensorFlow。
  → PyTorch, TensorFlow 같은 주류 프레임워크.

→ 영어 `such as` / `e.g.` 와 동일.

### `亦 / 也` — ~도 또한

- 该方法**也**适用于多模态场景。
  → 이 방법은 멀티모달 시나리오에도 적용 가능하다.

→ `亦`은 문어체, `也`는 일반체. 의미 동일.
