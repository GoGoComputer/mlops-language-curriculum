## 1. 왜 도메인 어휘부터 시작하는가 — 빈도 분포의 비밀

만약 여러분이 일반 중국어 학원에 가셨다면, 첫 수업에서 배우는 단어는 你好(니하오, 안녕하세요), 我(워, 나), 谢谢(셰셰, 감사합니다) 같은 거예요. 그 다음 단원에서는 吃饭(츨판, 밥 먹다), 学校(쉬에샤오, 학교), 朋友(펑요우, 친구) 같은 일상 어휘.

이 흐름은 **회화 학습 코스의 표준**입니다. 그리고 일상 회화를 목표로 한다면 정확한 흐름이에요.

그런데 잠깐 생각해보세요. 여러분의 목표가 ModelScope에서 Qwen3 모델 카드를 읽고, 知乎에서 vLLM OOM 답변을 찾는 거라면, 위 단어들이 거기 나올까요?

**거의 안 나옵니다.**

저는 실제로 작은 통계를 내봤어요. ModelScope의 Qwen3 모델 카드 첫 1,000자를 가져와서, 그 안의 단어들이 HSK(중국어 능력 시험) 각 급수와 얼마나 겹치는지 세봤습니다.

```
HSK 1급 (150 단어)         → 모델 카드와 겹치는 건 25개정도
HSK 2급 (300 단어 누적)    → 50개정도
HSK 3급 (600 단어 누적)    → 80개정도
HSK 4급 (1,200 단어 누적)  → 120개정도

그러면 모델 카드에는 나오는데 HSK 어디에도 없는 도메인 어휘는?
→ 200개 이상
```

즉 HSK 1급을 다 외워봤자 모델 카드의 12% 정도만 커버되고요, HSK 4급까지 1,200개를 다 외워도 모델 카드의 60-70%까지밖에 안 됩니다. 나머지 30-40%는 도메인 어휘로만 채워져 있고, 일반 학습 코스는 그걸 안 가르쳐줘요.

### 빈도 분포의 비대칭성

여기서 중요한 인사이트 하나. **언어의 단어들은 균등하게 분포하지 않습니다.** 자주 쓰이는 단어는 정말 자주 쓰이고, 안 쓰이는 단어는 정말 안 쓰여요. 이걸 통계학에서는 빈도 분포라고 부르고, 자연어에서는 보통 매우 가파른 곡선을 그립니다.

일상 중국어의 빈도 분포는 어떨까요? 약 1,500-2,000 단어를 알면 일상 회화의 80%를 이해할 수 있어요. 즉 일상 학습은 한 1,500개에서 일종의 임계 효과를 봅니다.

**그런데 MLOps 도메인 중국어는 분포가 훨씬 더 압축적입니다.** 약 200-300 단어만 알면 모델 카드의 80%를 이해할 수 있어요. 왜냐하면 도메인 어휘는 같은 단어가 미친 듯이 반복해서 나오기 때문이에요. 模型, 训练, 推理, 部署, 量化 — 이 5개 단어만 봐도 거의 모든 모델 카드에 등장합니다. 그것도 한 페이지에 몇 번씩.

**이 극단적 빈도 압축이 우리에게 주는 선물** — 220개만 정확히 외우면 모델 카드를 읽을 수 있어요. 일반 학습 코스에서 1년 동안 외우는 양보다 적은 양으로요. 한국어 화자라면 4개월. 영어 화자라면 6-8개월. 사용 가능한 학습 시간을 정확히 거기에 배분하는 게 우리 전략의 핵심입니다.

### 일반 학습 코스의 함정

만약 일반 학습 코스를 따라가면서 이 커리큘럼을 병행하면 어떻게 될까요? 답은: **둘 다 잘 안 됩니다.**

일반 학습 코스는 회화 위주라 듣기·말하기 연습이 큰 부분을 차지해요. 듣기·말하기는 시간 효율이 우리 목표(독해)에 비해 매우 낮습니다. 1시간 회화 연습으로 얻는 독해 진척은 1시간 모델 카드 독해의 1/5 정도밖에 안 돼요.

그래서 우리는 의도적으로 회화·듣기를 통째로 제외했어요. 죄책감 가지지 마세요. **목표가 다르면 경로가 달라야 합니다.** 회화는 졸업 후 6-12개월 별도 투자로 나중에 추가하면 됩니다.

## 2. 한자 학습 전략 — 어떻게 외울 것인가

자 이제 "도메인 어휘부터 시작한다"는 결론은 났습니다. 그럼 그 220개를 어떻게 외울까요? 단어 단위로 외울까요, 아니면 한자 단위로 외울까요?

답을 미리 드리면 — **단어 단위로 외우되, 그 과정에서 한자가 자동으로 학습됩니다.** 왜 그런지 풀어 드릴게요.

### 한자는 빌딩 블록

0단계 가이드에서 다뤘던 사실 한 가지가 결정적이에요. **한자는 빌딩 블록입니다.** 한 한자를 익히면 그 한자가 들어가는 다른 단어들도 동시에 점화돼요.

예를 들어 "推"라는 한자를 처음 만났다고 해봅시다. 0단계에서 봤듯 "扌(손) + 隹(새)"인데 "손으로 미는 동작" 뉘앙스를 가져요. 이걸 한 번 익히면:

```
推理 (추리)  →  이치를 미는 = 추론, inference
推论 (추론)  →  이론을 미는 = 추론
推断 (추단)  →  단정을 미는 = 추정/추론
推荐 (추천)  →  권하는 = 추천
推动 (추동)  →  움직이도록 미는 = 추진
```

5개 단어가 한 번에 점화됩니다. 추리는 ML에서 inference, 추천은 추천 시스템(recommendation system), 추진은 프로젝트를 진행시킨다는 의미. 모두 "推"의 의미가 살아 있어요.

또 하나의 예. "量"이라는 한자.

```
量化 (양화)  →  측정으로 변환 = quantization
增量 (증량)  →  증가하는 양 = increment
考量 (고량)  →  잘 살펴 측정 = 고려/검토
测量 (측량)  →  재고 측정 = 측정/measurement
```

이번엔 4개. "量"을 한 번 외우면 양자화, increment, 고려, 측정이 한꺼번에.

또 하나. "集".

```
集群 (집군)   →  무리의 모음 = cluster
数据集 (수거집)  →  데이터의 모음 = dataset
集成 (집성)   →  모아 이루다 = integration
集中 (집중)   →  모이다 = concentrate
```

**이게 한자의 마법이에요.** 한 글자 = 빌딩 블록. 빌딩 블록 100개를 모으면 단어 1,000개를 추측할 수 있게 돼요. 한자 학습이 알파벳 학습과 결정적으로 다른 점이 이거에요. 알파벳은 의미를 가지지 않지만, 한자는 그 자체가 의미 단위니까요.

### 한국어 화자의 두 번째 무기 — 한국 한자음

여러분이 한국어 화자라면 한자 학습에 추가 무기가 있어요. **한국어 한자음.** 0단계에서 잠깐 다뤘죠. 더 자세히 보면:

```
중국어 한자: 模型
중국어 발음:  mó xíng (모싱)
한국어 한자음: 모형
한국어 일반어: 모형 (= 모델)
중국어 의미:  모델
```

직접 매핑이에요. 한자가 같으면 한국어 한자음을 떠올리고, 그게 바로 의미입니다.

다른 예시 10개:

```
训练 → 훈련 → training ✓
部署 → 부서 → deployment ✓
量化 → 양화 → quantization ✓
微调 → 미조 → fine-tuning ✓
对齐 → 대제 → alignment ✓
蒸馏 → 증류 → distillation ✓
推理 → 추리 → inference (한국어로는 "추론"에 더 가까움)
模型 → 모형 → model ✓
参数 → 참수 → parameter ("참수"는 한국어 일반어 아님, "매개변수")
显存 → 현존 → VRAM ("현존"은 한국어 일반어 아님)
```

10개 중 7개는 한국어 한자음 그대로 의미가 잡히고, 3개(추리, 참수, 현존)는 한국어로는 잘 안 쓰는 한자어라 약간 추가 학습이 필요합니다. 그래도 70% 직접 매핑이에요.

영어 화자에게는 이 자산이 통째로 없어요. 그래서 영어 화자는 한자 빌딩 블록 전략으로만 4-6개월을 더 들여 학습해야 합니다.

### 그렇다면 어떤 한자부터 외울까?

답: **단어 220개를 외우는 과정에서 한자가 자동으로 학습됩니다.** 한자만 따로 외우지 마세요.

이 점이 중요해요. "한자 1,500자 책" 같은 걸 사서 따로 학습하지 마시고, 우리 vocab CSV를 단어 단위로 외우세요. 한자는 그 단어 안에 들어 있으니, 한 한자가 5-10개 단어에 등장하는 동안 자연스럽게 익숙해집니다.

예를 들어 "模"라는 한자는 다음 단어들에서 반복적으로 등장해요:
- 模型 (모델)
- 模板 (template)
- 模式 (pattern, mode)
- 规模 (scale)

이 4개 단어를 외우면 "模"라는 한자는 따로 외울 필요 없이 머릿속에 박힙니다. **한자 자체를 학습 단위로 삼는 건 비효율적이고, 단어를 학습 단위로 삼되 한자가 부산물로 학습되는 게 효율적이에요.**

## 3. 220개 도메인 어휘 카테고리

이제 220개 어휘가 어떻게 구성되어 있는지 카테고리별로 풀어드릴게요. 마스터 파일은 `vocab/mlops-domain-vocab.csv`이고, 다섯 개 카테고리로 나뉩니다.

### 카테고리 1: 모델·아키텍처 (70개+)

**이 카테고리가 다루는 것**: 모델 자체를 표현하는 어휘. 어떤 모델인지, 어떤 아키텍처인지, 어떤 학습 방법을 썼는지.

**왜 가장 많은가**: 모델 카드의 절반은 모델 자체에 대한 설명이거든요. 그래서 어휘도 가장 많이 모여요.

**핵심 한자 5개**:

```
模 (모) — 본/모형
型 (형) — 형태/타입
训 (훈) — 가르치다
练 (련) — 연습/단련
推 (추) — 밀다/추론하다
```

이 5 한자만으로 만들어지는 단어들:

| 단어 | 한국 한자음 | 의미 |
|------|-----------|------|
| 模型 | 모형 | model |
| 模板 | 모판 | template |
| 模式 | 모식 | pattern, mode |
| 规模 | 규모 | scale |
| 类型 | 류형 | type |
| 训练 | 훈련 | training |
| 训练集 | 훈련집 | training set |
| 预训练 | 예훈련 | pre-training |
| 微调 | 미조 | fine-tuning |
| 推理 | 추리 | inference |
| 推断 | 추단 | inference (다른 의미) |

**5개 한자에서 11개 단어.** 한 한자가 평균 2-3개 단어에 등장하는 구조에요. 5 한자를 외우면 11 단어가 풀리는 셈.

이 카테고리의 다른 핵심 어휘들 (한국어 한자음 매핑이 강한 것들):

```
架构 (가구) — architecture
参数 (참수) — parameter
神经网络 (신경망락) — neural network
注意力 (주의력) — attention
机制 (기제) — mechanism
权重 (권중) — weight
偏置 (편치) — bias
梯度 (제도) — gradient
损失 (손실) — loss
优化 (우화) — optimization
学习率 (학습율) — learning rate
正则化 (정칙화) — regularization
批量 (비량) — batch
轮次 (윤차) — epoch
迭代 (질대) — iteration
对齐 (대제) — alignment
蒸馏 (증류) — distillation
检索 (검색) — retrieval (RAG의 R)
增强 (증강) — augmented
```

처음 보면 막막하지만, 한국어 한자음이 거의 다 한국어 ML 용어와 같아요. "참수"는 한국어로 "매개변수", "학습율"은 그대로 "학습률", "정칙화"는 "정규화/규제화"로 매칭됩니다.

### 카테고리 2: 인프라·배포 (50개+)

**이 카테고리가 다루는 것**: 모델을 어떻게 배포하고 운영하는가. 컨테이너, 클러스터, GPU, 양자화 같은 인프라 단어들.

**핵심 한자 5개**:

```
部 (부) — 부분/배치
署 (서) — 배치/관청
容 (용) — 담다/용기
器 (기) — 기구
量 (량) — 양/측정
```

**대표 어휘**:

```
部署 (부서) — deployment
容器 (용기) — container
镜像 (경상) — image (Docker)
集群 (집군) — cluster
显存 (현존) — VRAM
显卡 (현카) — graphics card
量化 (양화) — quantization
压缩 (압축) — compression
推理引擎 (추리인경) — inference engine
推理框架 (추리광가) — inference framework
分布式 (분포식) — distributed
节点 (절점) — node
分片 (분편) — shard
副本 (부본) — replica
负载 (부재) — load
均衡 (균형) — balance (load balancer 같은 표현에서)
```

VRAM, GPU 같은 영어 약어는 그대로 영어로 쓰이는 경우도 많아요. 中文 자료에 "GPU 显存" 같이 한자와 영어가 섞이는 게 정상.

### 카테고리 3: 성능·평가 (20개+)

**이 카테고리가 다루는 것**: 성능 측정 어휘. throughput, latency, 정확도 같은 것들.

**왜 가장 적은가**: 카테고리는 작지만 빈도는 매우 높아요. 모델 카드의 벤치마크 섹션에 거의 다 등장하는 단어들이거든요.

**대표 어휘**:

```
吞吐量 (탄토량) — throughput
时延 (시연) — latency
延迟 (연지) — latency (다른 표현)
算力 (산력) — compute power
评测 (평측) — benchmark/evaluation
评估 (평고) — evaluation
基准 (기준) — baseline/benchmark
准确率 (준확률) — accuracy
召回率 (소회률) — recall
精确率 (정확률) — precision
错误率 (착오률) — error rate
速度 (속도) — speed
性能 (성능) — performance
对比 (대비) — comparison
```

"탄토량"은 한국어로 잘 안 쓰는 한자어인데, 직역하면 "삼키고 뱉는 양"으로 throughput 의미가 살아 있어요. 한 번 외우면 끝.

### 카테고리 4: 데이터·파이프라인 (25개+)

**이 카테고리가 다루는 것**: 데이터셋, 전처리, 토크나이저 같은 데이터 측면.

**대표 어휘**:

```
数据集 (수거집) — dataset
样本 (양본) — sample
标注 (표주) — annotation/labeling
标签 (표첨) — label/tag
预处理 (예처리) — preprocessing
分词 (분사) — tokenization
词表 (사표) — vocabulary
词汇 (사회) — vocabulary
编码 (편마) — encoding
解码 (해마) — decoding
管道 (관도) — pipeline
流水线 (유수선) — pipeline (다른 표현)
清洗 (청세) — cleaning
增广 (증광) — augmentation
切分 (절분) — split
划分 (획분) — divide/split
```

"管道"와 "流水线"은 둘 다 pipeline 의미로 쓰이고, 자료에 따라 다르게 나와요. 두 표현 다 알고 계셔야 헷갈리지 않습니다.

### 카테고리 5: 일반 (50개+)

**이 카테고리가 다루는 것**: 도메인 전용 단어는 아니지만 기술 문서에 매우 자주 등장하는 동사·연결어들.

**핵심 한자 6개**:

```
支 (지) — 지지하다
持 (지) — 가지다/유지하다
基 (기) — 기초/뿌리
于 (어) — ~에서
用 (용) — 쓰다
通 (통) — 통하다
```

**대표 어휘**:

```
支持 (지지) — supports
不支持 (불지지) — does not support
采用 (채용) — adopts
基于 (기우) — based on
通过 (통과) — through
适用 (적용) — applicable
适用于 (적용우) — applicable for
用于 (용우) — used for
可用 (가용) — available
需要 (수요) — needs/requires
要求 (요구) — requires
依赖 (의뢰) — depends on
实现 (실현) — implements
提供 (제공) — provides
包括 (포괄) — includes
包含 (포함) — contains
```

이 카테고리의 단어들은 강의 03 (문법 마스터)에서 다루는 15문형과 깊이 연결되어 있어요. 예를 들어 "用于", "基于", "通过"는 단어이면서 동시에 핵심 문형이에요. 그래서 어휘 학습과 문법 학습이 자연스럽게 같이 일어납니다.

### 220개 어휘를 어떻게 우선순위 정할까?

만약 시간이 정말 부족해서 220개 다 못 외우신다면, 다음 우선순위로:

1. **1순위 (50개)**: 카테고리 5의 일반 동사·연결어 모두. 이게 안 되면 문장 구조 자체가 안 잡혀요.
2. **2순위 (70개)**: 카테고리 1의 모델·아키텍처. 모델 카드의 핵심.
3. **3순위 (50개)**: 카테고리 2의 인프라·배포. 운영 측면.
4. **4순위 (20개)**: 카테고리 3의 성능·평가. 벤치마크 섹션.
5. **5순위 (25개)**: 카테고리 4의 데이터·파이프라인. 학습 측면.

그래도 가능하면 220개 다 하시기를 권장. 한 달이면 끝나는 양이에요.

## 4. Anki 셋업 — 단계별 가이드

이제 220개 어휘를 외울 도구를 셋업하겠습니다. 우리는 Anki라는 SRS(Spaced Repetition System, 간격 반복 학습) 도구를 씁니다. 무료고, 검증된 학습 알고리즘을 가지고 있어요.

### Step 1: vocab CSV 생성

저장소 루트에서:

```bash
python3 scripts/build_vocab.py
```

이 명령이 `vocab/build/` 폴더 아래에 언어별 CSV 파일을 만들어줍니다:

```
vocab/build/vocab-ko.csv  ← 한국어 학습자용 (Front: 한자, Back: 한국어 + 병음)
vocab/build/vocab-en.csv  ← 영어 학습자용
vocab/build/vocab-zh.csv  ← 병음만 (참고용)
```

한국어 화자는 `vocab-ko.csv`를 사용하시면 됩니다.

### Step 2: Anki 설치

[apps.ankiweb.net](https://apps.ankiweb.net/)에서 데스크톱 앱 다운로드. Windows/macOS/Linux 모두 무료. 모바일 앱은 Android는 무료 (AnkiDroid), iOS는 유료 (AnkiMobile, $25 한 번 결제). 모바일은 출퇴근 시간 활용에 매우 유용하니 iOS도 결제 추천.

### Step 3: CSV 가져오기

1. Anki 데스크톱 앱 실행
2. 새 덱 만들기 (이름: "MLOps 중국어")
3. 파일 → 가져오기 → `vocab/build/vocab-ko.csv` 선택
4. 가져오기 옵션:
   - 필드 매핑: Front → "front" 필드, Back → "back" 필드
   - 노트 타입: Basic (기본형)
5. 가져오기 실행 → 220개 카드 생성 확인

### Step 4: 일일 학습 양 결정

학습 페이스에 따라 신규 카드 수를 정합니다:

| 학습 페이스 | 일일 신규 카드 | 졸업까지 |
|-----------|---------------|----------|
| 표준 (4개월) | 30개/일 | 약 8일 (Anki 신규 220개 끝) |
| 가속 (3개월) | 50개/일 | 약 5일 |
| 천천히 (5-6개월) | 15개/일 | 약 15일 |

복습 카드 수는 무제한 (Anki가 자동 스케줄)으로 두세요.

### Step 5: 일일 루틴

```
아침 또는 출퇴근 길:
  10분 — Anki 신규 + 복습 (30개)

저녁:
  25분 — 모델 카드 한 단락 독해
  10분 — 모르는 단어 정리 + Anki 추가
```

같은 시간대에 매일 하시는 게 알고리즘에 맞아요. Anki의 간격 반복 알고리즘은 "일정한 시간 간격"을 가정하니까요.

### Step 6: 어려운 카드 처리

학습 중에 자꾸 까먹는 카드가 나옵니다. Anki에서 "Again" (다시) 또는 "Hard" (어려움) 버튼을 누르면 빨리 다시 등장해요. **무리해서 외우려고 하지 마시고**, 5번 이상 까먹는 카드는 잠시 일시 중지하시고 자료에서 자연스럽게 만나도록 두세요. 약 80%의 카드는 정상 학습되고, 까다로운 20%는 자료 노출로 채워집니다.

## 5. 한국 한자음 활용법 (한국어 화자 전용)

자, 한국어 화자 전용 섹션입니다. 영어 화자분들은 이 섹션을 건너뛰셔도 좋아요. 영어 화자용 가이드는 `docs/en/guide-02-vocabulary.md`에 따로 있습니다.

### 직접 매핑 — 의미 100% 일치

대부분의 도메인 어휘는 한국어 한자음 그대로 의미가 통합니다.

```
模型 → 모형 → model ✓
训练 → 훈련 → training ✓
部署 → 부서 → deployment ✓
量化 → 양화 → quantization ✓
微调 → 미조 → fine-tuning ✓
检索 → 검색 → retrieval ✓
对话 → 대화 → dialogue ✓
推理 → 추리 → inference (한국어 추리는 reasoning, 의미 비슷) ✓
```

### 의미 분기 — 한자는 같은데 뉘앙스가 다른 경우

```
支持 → 지지 → 한국어 의미 "지지"이지만 중국어에서는 "지원하다(support)"
机器 → 기계/기기 → 한국어와 비슷
处理 → 처리 → 한국어와 거의 같음
学习 → 학습 → 한국어 일반어 그대로
```

이런 경우에는 한국어 한자음의 의미를 약간 조정해서 사용하시면 돼요. "支持"를 "지지"로만 외우면 약간 어색하니, "지원/지지"로 동시에 기억하시는 게 좋습니다.

### 한국어에서 잘 안 쓰는 한자어

일부 단어는 한국어 한자음으로는 어색한 한자어예요. 이런 단어는 한국어 일반어와 별도로 학습이 필요합니다.

```
显存 → 현존 → VRAM (한국어 "현존"은 다른 의미)
显卡 → 현카 → graphics card
吞吐量 → 탄토량 → throughput (한국어에 없는 단어)
时延 → 시연 → latency
算力 → 산력 → compute power
召回率 → 소회률 → recall (한국어 "재현율"이 더 자연스러움)
```

**팁**: 이런 단어들은 한국 한자음을 통하지 말고 그냥 영어 의미로 직접 외우시는 게 효율적이에요. 한국어 한자음은 약 70%에서 도움 되고, 나머지 30%는 그냥 새 단어로 외우시면 됩니다.

### 함정 단어 — 의미가 갈라진 한자 조합

가장 조심해야 하는 그룹. 한자는 같은데 한국어와 중국어에서 의미가 완전히 다른 단어들. 약 30개 정도이고 자주 등장합니다.

| 중국어 | 한국 한자음 | 한국어 의미 (혹시 헷갈리는) | 중국어 실제 의미 |
|--------|------------|------------------------|------------------|
| 汽车 | 기차 | 기차 (기차역의 그 기차) | 자동차 (한국어 기차는 火车) |
| 工夫 | 공부 | 학습 | 시간/노력 (한국어 공부는 学习) |
| 新闻 | 신문 | 신문 (신문지) | 뉴스 (한국어 신문은 报纸) |
| 告诉 | 고소 | 법적 고소 | 알려주다 (말하다 의미) |
| 老婆 | 노파 | 늙은 할머니 | 아내 (구어체) |
| 学院 | 학원 | 사설 교육기관 | 단과대학 (한국어 학원은 培训班) |
| 食堂 | 식당 | 식당 (음식점) | 학교/회사 식당 (음식점은 餐厅) |
| 经理 | 경리 | 회계 | 매니저/관리자 (한국어 경리는 会计) |
| 写真 | 사진 | 사진 | 실물 그림 (한국어 사진은 照片) |
| 床 | 상 | 평상/상 | 침대 |
| 走 | 주 | 달리다 | 걷다 (달리다는 跑) |
| 妈妈 | 마마 | 임금/비속어 | 엄마 |

기술 문서에는 위 함정 단어 중 몇 개만 등장하는 편이에요 (老婆, 食堂 같은 건 모델 카드에 안 나옴). 자주 등장하는 건 走, 经理 정도. 자료에서 만나면서 자동으로 수정됩니다.

### 한국 한자음이 약한 한국어 화자라면?

요즘은 한자 학습을 안 한 한국어 화자도 많아요. 그런 분이라면 다음 보완 전략을 권장합니다.

1. **vocab CSV에 한자 표시가 있으니 그걸 학습 단위로 삼기**: Anki 카드의 Front에 한자, Back에 한국어 한자음 + 의미가 같이 나옵니다. 한자음을 자연스럽게 익히게 됩니다.
2. **한국 한자음 인기 책 한 권 읽기**: 시중에 "한자 1,800자" 같은 책이 있어요. 이걸 처음 한 달 동안 30분씩 같이 보시면 한자음 직관이 빠르게 살아납니다.
3. **포기하지 말기**: 한국 한자음이 약해도 시작할 수 있어요. 처음 50개 어휘만 외우면 한자음이 다시 살아나기 시작합니다.

## 6. "외우려고 하지 말고 만나라" — 빈도 학습의 원리

여기서 이 커리큘럼의 가장 중요한 원칙을 풀어 드리겠습니다. 220개를 다 외운 후 자료를 읽기 시작하면, 새 단어가 미친 듯이 등장해요. 자연스러운 반응: "더 외워야지!" → 잘못된 반응.

### 왜 외우면 더 잘 까먹는가? (역설)

여러분이 새 단어 100개를 강제로 Anki에 추가했다고 합시다. 일주일 후 적중률은 어떻게 될까요? 보통 50-60%로 떨어집니다. 왜냐하면:

1. **Anki 알고리즘은 빈도 정보를 모릅니다.** 자주 만나는 단어와 안 만나는 단어를 똑같이 다뤄요.
2. **자주 안 만나는 단어는 컨텍스트가 약합니다.** 컨텍스트 없는 단어는 마치 무작위 비밀번호처럼 외우기 어렵죠.
3. **단어 100개를 추가하면 복습 부담이 폭증합니다.** 결과적으로 신규 220개와 기존 어휘까지 다 약해져요.

### 빈도 학습의 메커니즘

자연스러운 빈도 학습은 다음과 같이 작동합니다:

```
자료에서 새 단어 만남 (1회)
  → Pleco 검색 → 의미 확인 → 메모 → 넘어감

같은 단어 다시 만남 (2회, 며칠 후)
  → "어, 이거 본 적 있는데?" → Pleco 한 번 더 → 메모 강화

3회째 만남
  → "아 그 단어다" → 검색 안 해도 됨

4-5회째 만남
  → 자동으로 의미가 떠오름. 외워진 상태.
```

**이 흐름의 핵심**: 단어를 외우려고 노력하지 않고, 자료에서 자연스럽게 만나면서 학습됩니다. 자주 만나는 단어 = 중요한 단어 = 자동으로 외워짐. 안 만나는 단어 = 안 중요한 단어 = 굳이 외울 필요 없음. 이게 우리 뇌가 모국어를 학습한 방식과 똑같아요. 모국어는 "외운" 게 아니라 "수만 번 만나면서" 외워진 거잖아요.

### 그러면 220개는 왜 외우는가?

220개는 **임계점을 넘기기 위한 시드(seed)**예요. 220개를 외우면 모델 카드의 80%가 풀려요. 80%가 풀리면 이해하면서 읽을 수 있고, 이해하면서 읽으면 새 단어를 컨텍스트 안에서 만나게 돼요. 컨텍스트 안에서 만나면 빈도 학습이 작동하기 시작하고, 그 다음부터는 자동입니다.

220개 = 빈도 학습이 작동하기 시작하는 임계점. 그 아래로는 안 풀려요. 그래서 의도적으로 220개를 강제 외우는 거에요.

### 자료에서 새 단어 만났을 때 처리 워크플로우

```
[1단계] 한자 분해 — 부수 + 한국 한자음
   예: 实际上 → 实(실) + 际(제) + 上(상) → "실제상"

[2단계] 의미 추측
   "실제상" → 한국어로는 "실제로" 비슷한 의미

[3단계] Pleco 검증
   shí jì shàng → "in fact, actually"

[4단계] 메모만 (외우지 않음)
   노트북에 한 줄 적기. Anki에 추가하지 않음.

[5단계] 다음 문장으로 넘어감
   3-5번 다시 만나면 자동으로 외워짐.
```

### 어떤 단어를 Anki에 추가하고 어떤 걸 안 할지

기본 룰: **자료에서 만난 단어는 Anki에 추가하지 마세요.** 빈도 학습으로 처리하세요.

예외:
- **새로 등장한 도메인 약어** (RLHF, MoE, DPO, GRPO 같은 것). 이건 아직 도메인 어휘 표준에 없으니 추가.
- **자기 분야 전용 어휘** (자율주행이라면 LiDAR 관련, 멀티모달이라면 vision 관련). 이건 220개가 안 다루니 추가.
- **3번 이상 자료에서 만났는데 매번 까먹는 단어**. 이건 빈도 학습이 실패한 신호니 강제 외움 모드 ON.

이런 예외는 한 달에 30개 정도. 부담 안 됩니다.

| Anki 누적 카드 | 220 | 400 | 700 |
| Anki 적중률 | 80%+ | 80%+ | 80%+ |
| 모델 카드 1단락 시간 | 30분 | 15분 | 5분 |
| DeepL 없이 이해도 | 30% | 60% | 80% |
| 자료에서 만나는 새 단어 비율 | 70%+ | 30% | 10% |

**측정 미달 시 대응**:
- Anki 적중률 70% 미만 → 다음 주 신규 카드 줄이고 복습 강화
- 독해 시간이 줄지 않음 → 강의 03(문법) 가이드 다시 통독
- 새 단어 비율이 안 줄음 → 다른 도메인 자료로 갈아타기 (자기 분야로)

### 자가 진단 예시 시나리오

**시나리오 A**: Week 4에 Anki 카드 220개 다 봤는데 적중률 65%. 어떡하나?

대응: 신규 카드 일시 중지. 일주일 동안 복습만. Anki 알고리즘이 약한 카드를 다시 띄울 거예요. 그게 끝나면 적중률이 80%+로 회복됩니다.

**시나리오 B**: Week 8인데 모델 카드 한 단락 읽는데 여전히 25분. 정상인가?

판단: 약간 느림. 신규 어휘 학습보다 **노출이 부족한 게 원인**일 가능성 높음. 모델 카드 한 단락 매일 1편씩 늘리세요. 노출이 빈도 학습을 가속합니다.

**시나리오 C**: Week 12인데 슬럼프. 진척이 안 보임.

대응: 일주일 휴식. 또는 Stellaris 中文 위키, 사주 中文 글 같은 비기술 자료로 일주일 읽기. 도메인 외부 노출이 의외로 큰 도움 됩니다.

## 8. 슬럼프 대응

학습 슬럼프는 보통 Week 5-6 (Month 2 초반)에 옵니다. 이유: 도메인 어휘는 다 외웠는데, 실제 자료에서 막히는 단어가 너무 많이 보이기 시작.

이건 사실 진척이 멈춘 게 아니에요. **임계점을 넘는 과정에서 보이는 정상적 현상**이에요. 220개에서 다음 단계(Month 2 도메인 어휘 추가 100개)로 넘어가는 사이에 잠깐 우물에 빠지는 느낌이 듭니다.

### 슬럼프 대응 3가지

**1. 친숙한 분야로 갈아타기**

기술 문서에서 잠깐 벗어나세요. 자기에게 친숙한 분야의 中文 자료로 일주일.

- 게임 좋아하시면 Stellaris, MTG, Starcraft 等 中文 위키
- 사주·역학 좋아하시면 中文 명리학 글
- 음식 좋아하시면 微博의 음식 후기
- 영화 좋아하시면 豆瓣의 영화 리뷰

이런 비기술 자료에서 한자 노출을 늘리면, 다시 기술 자료로 돌아갔을 때 어휘가 더 잘 박혀요.

**2. 목표 재확인**

4개월 후 GitHub Issue 1건. 그 1건만 쓰면 졸업이에요. 그 1건의 모습을 구체적으로 상상해보세요. 어떤 OSS에, 어떤 문제로, 어떤 톤으로? 이미지가 명확해지면 슬럼프가 쉽게 풀려요.

**3. 첫 단락 리뷰**

한 달 전에 막혔던 모델 카드 단락을 다시 읽어보세요. 30분 걸리던 게 5분에 끝나는 걸 체감하시면, 진척이 진짜 있다는 증거가 손에 잡혀요.

### 절대 하지 말아야 할 것

- **220개를 처음부터 다시 외우기**: 시간 낭비. 약한 카드만 Anki가 다시 띄워줍니다.
- **다른 학습 코스로 갈아타기**: 일주일이면 갈아타다가 시간만 낭비. 슬럼프는 며칠이면 끝납니다.
- **포기하기**: Week 5-6은 가장 흔한 포기 시점. 한 주만 더 버티면 임계점을 넘어요.
