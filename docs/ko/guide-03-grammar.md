## 0. 이 가이드를 어떻게 읽을 것인가

이 가이드는 다음 흐름으로 진행됩니다.

먼저 **왜 문법 15개로 충분한가**라는 질문에 답을 드립니다. 이게 사실 가장 중요한 부분이에요. 이 답을 받아들이지 못하시면 학습 동기가 흔들립니다.

그 다음에 **중국어 어순의 기본**을 잠깐 다룹니다. 0단계에서 약간 다뤘지만, 본격 문장 학습에 필요한 만큼 더 깊이.

그리고 **핵심 5문형**을 자세히 풀어드려요. 각 문형마다 정의 + 5-7개 예문 + 변형 + 함정. 이 5개만 알아도 모델 카드의 절반은 풀립니다.

마지막으로 **추가 10문형**과 **실전 적용**을 다룹니다. 그러면 80% 커버. 그 위는 자료 노출로 자동.

외울 건 없어요. 이해하면서 읽으시고, 그 다음 자료에서 만나면서 자동화시키시면 됩니다.

## 1. 회화 문법 vs 독해 문법 — 사실은 다른 영역입니다

먼저 이상한 사실 하나 말씀드릴게요.

여러분 영어 회화 잘하시나요? 한국 개발자 대부분이 "아니요"라고 답하실 거예요. 그럼 다음 질문 — 영어 README 읽으시나요? 대부분 "네"라고 답하실 거예요.

같은 사람인데 이 두 답이 어떻게 양립할까요?

답: **회화 문법과 독해 문법은 사실상 다른 영역**입니다. 두 영역의 겹침은 생각보다 작아요.

영어 회화에 필요한 건 일상 표현, 의문문 도치, 시제 변화의 정확한 발화, 관용 표현 등이에요. 영어 독해에 필요한 건 문장 골격(SVO + 수식어), 정형 표현(it is X that..., given that..., provided that...), 그리고 도메인 어휘 + 문맥 추론. 둘 다 영어이지만 학습해야 하는 항목이 다르고, 그래서 한쪽만 잘 해도 다른 쪽은 못 할 수 있어요.

### 중국어로 옮기면

중국어 회화에서 가장 중요한 문법 그룹 하나가 있어요. **어기 조사(语气助词).** 한국어로 정확히 대응하는 게 없어서 학습자가 가장 힘들어하는 영역입니다. 대표 4개:

```
吧 (ba) — 부드러운 권유/제안
   你来吧 → "(좀) 와봐"

啊 (a) — 감탄/강조
   你来啊 → "와아 / 너도 좀 와"

呢 (ne) — 의문/대조
   你来呢? → "너는 (어떻게) 올 거야?"

嘛 (ma) — 당연/볼멘소리
   你来嘛 → "(좀) 오라고 / 와줘"
```

비슷해 보이는데 톤이 미묘하게 달라요. 정확히 구사하는 건 회화 학습의 핵심이고, 외국인이 가장 힘들어하는 부분이에요.

**그런데 ModelScope 모델 카드에 吧 啊 呢 嘛가 등장할까요?**

거의 안 나옵니다. 왜냐하면 기술 문서는 화자의 감정·뉘앙스를 전달하는 게 아니라, **사실을 객관적으로 전달**하는 게 목적이거든요. 어기 조사는 정의상 화자의 감정을 담는 도구라서, 객관적 사실 전달에는 안 맞아요. 모델 카드, 논문, README, GitHub Issue, Zhihu 답변 — 어디에도 거의 안 등장합니다.

이게 무슨 말이냐. **회화 문법의 가장 핵심적인 부분이 우리 학습에서는 0% 우선순위**라는 거예요. 이걸 안 배워도 모델 카드는 읽을 수 있어요.

### 그럼 독해 문법은 무엇이 핵심인가

독해 문법, 즉 기술 문서에 자주 등장하는 문형은 다음 5개가 절반을 커버합니다:

```
把 / 将 — 처분문 (X를 Y로 처리한다)
用于 — "에 사용된다"
基于 — "기반으로"
对...进行 — "에 대해 ~를 수행한다"
```

이 5개는 회화에서는 거의 안 쓰여요 (어색하거나 너무 격식적). 그래서 회화 코스에서는 4-5급 이후에 배워요. 우리 커리큘럼에서는 첫 주에 배웁니다.

**여기서 5개를 추가합니다**: 是...的(강조), 不/没(부정), 可以/能(가능), 如果...就(조건), 通过(수단). 그리고 또 5개: 需要(필요), 支持(지원), 被(피동), 연동문(동사 두 개 연결), 得(정도 보어). 합쳐서 15개. 이게 모델 카드와 README의 80%를 커버합니다.

15개. 4개월짜리 학습에서 4주에 끝낼 양이에요.

## 2. 중국어 어순 — 좋은 소식과 나쁜 소식

### 좋은 소식: SVO

중국어 어순은 **주어 + 동사 + 목적어**(SVO)입니다. 영어와 같아요.

```
我      训练    模型
나      훈련    모델
(I)     (train) (the model)
```

영어와 일치하는 어순이라, **영어를 읽으시는 한국 개발자라면 영어 어순 직관을 그대로 가져오시면 됩니다.** "I train the model" 머릿속에서 만들고, 그걸 한자로 옮기는 흐름.

### 나쁜 소식: 한국어 SOV와 다름

한국어는 **주어 + 목적어 + 동사**(SOV)이죠.

```
나는    모델을    훈련시킨다
(I)     (model)   (train)
```

그래서 한국어 화자분이 처음 중국어를 만나시면 어순이 어색해요. "나는 + 훈련 + 모델" 같이 동사가 가운데 들어가는 게 자연스럽지 않거든요.

**해결책**: 외우는 게 아니라 **노출로 익히는 거예요.** 모델 카드 한 단락을 매일 한 편씩 한 달 동안 읽으면, 어순은 자연스럽게 자동화됩니다. 영어를 평소에 읽으시는 분이라면 더 빨라요.

### 시제 표시 방법

중국어는 동사가 활용되지 않습니다. 시제는 보조 단어로 표시해요.

| 보조 | 의미 | 예 |
|------|------|-----|
| 了 (le) | 완료 | `我训练了模型` 나는 모델을 훈련시켰다 |
| 过 (guo) | 경험 | `我训练过 GPT` 나는 GPT를 훈련시켜본 적 있다 |
| 在 / 正在 (zài) | 진행 | `我在训练模型` 나는 모델을 훈련시키고 있다 |
| 将 (jiāng) | 미래/예정 (문어체) | `将发布新版本` 새 버전을 발표할 예정 |

영어의 -ed, -ing, will 같은 역할이에요. 동사 자체는 변하지 않고, 보조 단어가 시제를 알려줍니다.

### 수식어 위치

명사 앞: `大模型` (큰 모델). 영어 어순과 일치 (large model).

시간/장소 앞: `今天我训练模型` (오늘 나는 모델을 훈련시킨다). 한국어와 비슷.

피수식어 + 的 + 수식 절: `开源的模型` (오픈소스인 모델). 영어 관계절과 비슷한 위치.

## 3. 핵심 5문형 — 모델 카드의 50% 커버

자, 이제 진짜 핵심으로 들어갑니다. 이 5개 문형은 **모델 카드 한 단락에서 평균 5-10번 등장**해요. 이걸 자동 인식할 수 있게 되면 모델 카드 절반이 풀립니다.

각 문형마다 정의 → 예문 → 변형 → 함정 순서로 풀어드립니다.

---

### 3.1 把 (bǎ) — 처분문

**의미**: "X를 (어떻게) 처리한다"

영어로 치면 "convert X to Y", "load X into Y" 같은 패턴이에요. 무엇을 어떻게 했는지를 명확히 하는 구조.

**구조**:

```
주어 + 把 + 목적어 + 동사 + 보충(어디로/무엇으로/어떻게)
```

**예문 7개**:

1. `把模型转换为 ONNX 格式` — 모델을 ONNX 형식으로 변환한다
2. `把数据加载到内存` — 데이터를 메모리에 로딩한다
3. `把权重保存到 checkpoint` — 가중치를 체크포인트에 저장한다
4. `把模型部署到 GPU` — 모델을 GPU에 배포한다
5. `把输入序列截断为 512` — 입력 시퀀스를 512로 자른다
6. `把损失函数定义为交叉熵` — 손실 함수를 교차 엔트로피로 정의한다
7. `把这些层冻结` — 이 레이어들을 동결한다

**왜 이 문형이 자주 나오는가**

기술 문서는 "무엇을 무엇으로 처리했는가"를 명확히 전달해야 해요. 把는 이걸 강조하는 구조입니다. 단순히 "我训练模型"(나는 모델을 훈련한다)이 아니라, "我把模型转换为 ONNX"(나는 모델을 ONNX로 변환한다) — 처리 결과까지 명확히.

**중요한 함정 — 把 다음에 오는 동사는 단독으로 못 옴**

把 처분문에서 동사 뒤에는 반드시 **보충(complement)**이 있어야 해요. 어디로(到), 무엇으로(为/成), 어떻게(完了, 完成) 같은 거.

```
✗ 把模型训练       — 어색. 보충 없음.
✓ 把模型训练完了    — 모델 훈련을 완료했다 (보충: 完了)
✓ 把模型训练成最佳状态 — 모델을 최적 상태로 훈련했다 (보충: 成最佳状态)
```

**变化형: 한국어 직역의 함정**

把 처분문을 한국어로 직역하면 "을/를 X시켰다"가 되는데, 한국어로는 "을 시키다" 같은 표현이 너무 일반적이라 把의 미묘한 뉘앙스를 잃기 쉬워요. 한국어 화자분은 把가 등장하면 "어떻게 처리했는지"가 중요한 정보다라는 신호로 받아들이세요.

---

### 3.2 将 (jiāng) — 把의 문어체

**의미**: 把와 사실상 같음. 다만 격식이 더 높아요.

기술 문서, 논문, 공식 문서에서는 把보다 將을 더 자주 씁니다. 모델 카드의 90%는 把가 아닌 將을 써요. 캐주얼한 README나 블로그는 把를 더 자주 쓰고요.

**구조**: 把와 동일.

```
주어 + 将 + 목적어 + 동사 + 보충
```

**예문 5개**:

1. `将参数初始化为 0` — 매개변수를 0으로 초기화한다
2. `将模型量化至 INT4` — 모델을 INT4로 양자화한다
3. `本节将介绍核心算法` — 본 절에서 핵심 알고리즘을 소개한다
4. `将权重转换为 fp16` — 가중치를 fp16으로 변환한다
5. `将训练时间缩短到 3 天` — 훈련 시간을 3일로 단축한다

**구별 기준**: 모델 카드/논문 → 將. 캐주얼 README → 把. 두 개 다 알고 있으면 됩니다.

**將의 또 다른 용법 — 미래 시제**

將은 "예정"이라는 의미도 있어요. 把는 이 의미가 없습니다.

```
将发布 v3 版本 — v3 버전을 발표할 예정
将支持长上下文 — 긴 컨텍스트를 지원할 예정
```

이때 將은 영어의 "will" 또는 "shall"과 비슷해요. 처분문에 쓰일 때는 把와 같은 의미고, 동사 앞에 단독으로 쓰일 때는 미래 시제라는 점만 구별하시면 됩니다.

### 3.3 用于 (yòng yú) — "에 사용된다"

**의미**: A는 B에 사용된다 / A는 B를 위한 것이다

**구조**:

```
주어 + 用于 + 용도/목적
```

**예문 7개**:

1. `该模型用于文本生成` — 이 모델은 텍스트 생성에 사용된다
2. `该框架用于推理加速` — 이 프레임워크는 추론 가속에 사용된다
3. `本工具用于调试` — 본 도구는 디버깅에 사용된다
4. `适用于多种场景` — 다양한 시나리오에 적용 가능 (适用于 변형)
5. `可用于工业部署` — 산업 배포에 사용 가능 (可用于 변형)
6. `专用于科学计算` — 과학 계산 전용 (专用于 변형)
7. `主要用于研究` — 주로 연구에 사용된다

**왜 이 문형이 모델 카드의 첫 줄에 항상 등장하는가**

모델 카드의 핵심 정보 중 하나가 "이 모델이 무엇을 위한 것인가"예요. `用于` 또는 그 변형이 그 정보를 가장 자연스럽게 전달합니다. 그래서 모델 카드의 두 번째 문장 안에 99% 등장해요.

**우선순위**: 외우는 우선순위 1번. 첫 주에 외우세요.

---

### 3.4 基于 (jī yú) — "기반으로"

**의미**: A 기반의 B / A를 토대로 한 B

**구조**:

```
基于 + A + 的 + B  (A 기반의 B)
基于 + A + 동사       (A를 기반으로 ~한다)
```

**예문 7개**:

1. `基于 Transformer 架构` — Transformer 아키텍처 기반
2. `基于 LoRA 微调` — LoRA 기반 미세조정
3. `基于注意力机制的模型` — 어텐션 메커니즘 기반의 모델
4. `基于强化学习的对齐` — 강화학습 기반의 정렬
5. `基于 PyTorch 实现` — PyTorch 기반 구현
6. `基于 fp8 的训练方案` — fp8 기반의 훈련 방안
7. `本系统基于 Kubernetes 构建` — 본 시스템은 Kubernetes 기반으로 구축된다

**왜 모델 소개의 정형구인가**

새 모델을 소개할 때 가장 먼저 답해야 하는 질문이 "어떤 아키텍처/기법을 기반으로 하는가?"예요. `基于`가 그 답변의 표준 표현입니다. 모델 카드의 세 번째 문장 안에 거의 등장해요.

**우선순위**: 외우는 우선순위 2번.

---

### 3.5 对...进行 (duì...jìnxíng) — "에 대해 ~를 수행"

**의미**: A에 대해 B를 수행한다

영어로 치면 "perform Y on X", "apply Y to X"

**구조**:

```
对 + A + 进行 + B
   (A에 대해)  (B를 수행한다)
```

**예문 7개**:

1. `对模型进行量化` — 모델에 대해 양자화를 수행한다
2. `对参数进行初始化` — 매개변수에 대해 초기화를 수행한다
3. `对数据进行预处理` — 데이터에 대해 전처리를 수행한다
4. `对结果进行评估` — 결과에 대해 평가를 수행한다
5. `对损失进行优化` — 손실에 대해 최적화를 수행한다
6. `对模型进行 INT8 量化` — 모델에 대해 INT8 양자화를 수행한다
7. `对长序列进行截断` — 긴 시퀀스에 대해 자르기를 수행한다

**왜 이 문형이 진짜 빈출인가**

기술 문서에서 "어떤 객체에 대해 어떤 작업을 했다"라고 표현할 때 가장 자연스러운 구조거든요. `量化模型`(모델을 양자화한다)도 가능하지만, `对模型进行量化`(모델에 대해 양자화를 수행한다)가 격식 있는 표현이라 모델 카드/논문에서 압도적으로 많이 써요.

**한 모델 카드에 5-10번 등장하는 게 일반적입니다.** 자동 인식이 되어야 해요.

**한국어 직역 함정**: `对X进行Y`를 "X에 대해 Y를 수행한다"로 직역하면 한국어로는 어색해요. 자연스러운 한국어로 옮기면 "X를 Y한다"입니다. 즉 `对模型进行量化` = "모델을 양자화한다"로 의미를 잡으세요. 단어 단위 직역에 매달리지 마세요.

## 4. 추가 10문형 — 80% 커버까지

핵심 5문형이 50%를 커버한다면, 다음 10문형이 추가로 30%를 더해서 합계 80%까지 도달합니다.

### 4.1 是...的 (shì...de) — 강조문

**의미**: 강조 구조. "X는 Y인 것이다."

```
该模型是基于 Transformer 的 — 이 모델은 Transformer 기반인 것이다
该方法是有效的 — 이 방법은 효과적이다 (강조)
```

기술 문서에서는 "이런 특징이 있는 것이다"라는 강조 톤으로 자주 쓰여요.

### 4.2 不 / 没 (bù / méi) — 부정

**중요한 구별**:

| 부정사 | 사용 시점 | 예 |
|--------|-----------|-----|
| 不 | 미래·습관·상태 부정 | 不支持 (지원하지 않는다) |
| 没 | 과거·완료 부정 | 没训练 (훈련하지 않았다) |

기술 문서에는 不이 압도적으로 더 많이 나와요. 没는 "아직 안 했다" 류 표현에서.

```
不支持 fp8 — fp8을 지원하지 않는다
不需要预训练 — 사전 훈련이 필요 없다
没经过对齐 — 정렬을 거치지 않았다
```

### 4.3 可以 / 能 (kěyǐ / néng) — 가능

**의미 차이는 미세함. 둘 다 "할 수 있다"로 읽으세요.**

미묘한 차이:
- `可以` — 허용/가능 (general possibility, permission)
- `能` — 능력 (ability, capacity)

```
可以使用 INT4 — INT4를 사용할 수 있다 (가능)
能加载 70B 模型 — 70B 모델을 로딩할 수 있다 (능력)
```

기술 문서에서는 둘 다 자주 등장하고, 의미를 엄밀히 구별 안 해도 큰 무리 없어요.

### 4.4 如果...就 / 那么 (rúguǒ...jiù) — 조건문

**의미**: 만약 X라면, Y

```
如果显存不够,就使用量化 — 만약 VRAM이 부족하면, 양자화를 사용해라
如果是 fp8,那么需要 H100 — 만약 fp8이라면, H100이 필요하다
若不支持 fp8,使用 fp16 — fp8을 지원하지 않으면, fp16을 사용해라
```

`若`는 `如果`의 문어체. 더 격식 있는 문서에 등장.

### 4.5 通过 (tōngguò) — "~를 통해"

**의미**: A를 통해 B한다 / A를 수단으로 B한다

```
通过 LoRA 微调 — LoRA를 통해 미세조정
通过 API 调用 — API를 통해 호출
通过对模型进行量化,显存减少 75% — 모델에 대해 양자화를 수행하여 VRAM이 75% 감소
```

핵심 5문형의 `对...进行`과 자주 결합되어 등장해요. `通过对X进行Y` 같은 복합 구조.

### 4.6 需要 (xūyào) — "필요하다"

**의미**: ~가 필요하다

```
需要 80GB 显存 — 80GB VRAM이 필요하다
需要 PyTorch 2.0 以上 — PyTorch 2.0 이상이 필요하다
需要 GPU 支持 — GPU 지원이 필요하다
```

요구사항 섹션의 표준 단어. 모델 카드의 사양 부분에 매번 등장.

### 4.7 支持 (zhīchí) — "지원하다"

**의미**: ~를 지원한다

```
支持 INT4 量化 — INT4 양자화를 지원한다
不支持流式输出 — 스트리밍 출력을 지원하지 않는다
仅支持英文 — 영어만 지원한다
```

기능 명세의 표준 단어. 모델 카드의 첫 단락에 보통 등장.

### 4.8 被 (bèi) — 피동문

**의미**: ~된다 / ~받는다

```
模型被部署到 GPU — 모델이 GPU에 배포된다
参数被初始化为 0 — 매개변수가 0으로 초기화된다
该方法被广泛采用 — 이 방법은 광범위하게 채택된다
```

기술 문서의 피동문은 영어보다 적게 등장해요. 그래서 만나면 강조 의미가 살짝 들어 있다고 느끼시면 됩니다.

### 4.9 연동문 (V1 + V2) — 동사 두 개 연결

**의미**: 동사 두 개를 이어 쓰는 구조. 첫 동사가 두번째 동사의 수단/목적/방법을 표현.

```
使用 Python 训练模型 — Python을 사용하여 모델을 훈련한다
下载模型部署到本地 — 모델을 다운로드하여 로컬에 배포한다
打开终端运行命令 — 터미널을 열고 명령을 실행한다
```

영어로 치면 "use X to do Y" 또는 "do X and Y" 패턴. 기술 문서의 사용법 섹션에 매우 자주.

### 4.10 得 (de) — 정도 보어

**의미**: 동사가 얼마나 ~하게 진행되는가를 보충

```
性能提升得很快 — 성능이 매우 빠르게 향상된다
训练得不充分 — 충분히 훈련되지 않았다
读得很流畅 — 매우 매끄럽게 읽힌다
```

영어 -ly 부사 또는 "so much that..." 정도의 역할. 성능 비교 표현에서 자주 등장.

## 5. 실전 적용 — 모델 카드 한 단락 분석

이제 학습한 15문형이 실제 모델 카드에서 어떻게 작동하는지 시연해드리겠습니다.

**원문**:

```
该模型基于 Transformer 架构,采用 MoE 设计,可用于多种推理场景。
通过对模型进行 INT4 量化,显存占用减少 75%。
支持长上下文,需要至少 80GB 显存。
```

**문형별 식별**:

| 부분 | 문형 | 노트 |
|------|------|------|
| `基于 Transformer 架构` | 4번 (基于) | "Transformer 아키텍처 기반" |
| `采用 MoE 设计` | 동사 + 목적어 (기본) | "MoE 설계를 채택" |
| `可用于多种推理场景` | 3번 (用于 변형) | 可 + 用于 = "사용 가능" |
| `通过对模型进行 INT4 量化` | 4.5번 + 5번 (通过 + 对...进行) | "모델에 대해 양자화를 수행하여" |
| `显存占用减少 75%` | 동사 + 목적어 + 보충 | "VRAM 사용량이 75% 감소" |
| `支持长上下文` | 4.7번 (支持) | "긴 컨텍스트 지원" |
| `需要至少 80GB 显存` | 4.6번 (需要) | "최소 80GB VRAM 필요" |

**통합 번역**:

> 이 모델은 Transformer 아키텍처 기반이며, MoE 설계를 채택했고, 다양한 추론 시나리오에 사용될 수 있다. 모델에 대해 INT4 양자화를 수행하여 VRAM 사용량이 75% 감소한다. 긴 컨텍스트를 지원하며, 최소 80GB VRAM이 필요한다.

**한 단락에 7개의 문법 패턴이 등장**했어요. 그중 5개가 우리 핵심 15문형 안에 있고요. 이걸 자동 인식할 수 있게 되면 모델 카드 독해 속도가 폭발적으로 빨라집니다.
